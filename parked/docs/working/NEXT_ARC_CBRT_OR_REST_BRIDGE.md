## The one clean disjoint pickup I do see: an exact `Fixed::cbrt`

In the determinism-and-numerics forte, verifiable on the CPU here, disjoint from A's ledger and B's physics lanes, there is one clean primitive the floor is missing. Today every cube root on the floor is `powf(1/3)`, the exponential of a third of the logarithm, so it carries the compounded rounding of two transcendental series where the square root, by contrast, is a single exact integer `isqrt` (`fixed.rs:217`) with no series error at all. There is no `cbrt` sibling. An exact `Fixed::cbrt` is the natural one: the result bits are `icbrt(bits << 2*FRAC_BITS)` by the same identity the square root uses (`cbrt(b / 2^F) * 2^F = icbrt(b * 2^(2F))`), the radicand fits a `u128` (a positive `i64` shifted by sixty-four bits stays under `2^128`), and the integer cube root is a short deterministic binary search with no float and no series, bit-identical on every machine and thread count, exactly the square root's determinism contract.

Its value is real and specific. It is bit-exact where `powf(1/3)` is series-approximate, so it tightens the Rayleigh boundary-layer length and any other one-third power on the floor. It is a member of the exact-root family the arming's exponent field is scoped around (the exact-root exponents, a half and a composable cube root), so building the exact cube root now hands that field its primitive rather than leaving it on the powf approximation. And adding it is byte-neutral: a new primitive that no scenario calls changes no canonical state, and the existing `powf(1/3)` callers stay untouched, so the four `run_world` pins hold at `40fe8a72`; whether to rewire any caller onto it is a later, separately-gated call, not part of adding the primitive. It is provable here without hardware: the cube of the result reproduces the input to within the last representable bit, the result matches `powf(1/3)` to the powf precision target, and the perfect cubes (one, eight, twenty-seven) come back exact.

The honest limit: this is a precision-and-cleanliness gain, not a capability the floor lacks (`powf(1/3)` already computes cube roots deterministically). So it is worth building if the gate wants the exact-root family completed and the one-third powers tightened, and it is fair to defer if the gate would rather I rest for the retire. That is the gate's call, which is why I bring it rather than build it unasked.
