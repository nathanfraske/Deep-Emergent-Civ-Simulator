# Arc 6 (grow biosphere bodies from genome) build plan

From the ultracode scoping workflow (2026-07-08, map + high-effort design, verified against source). Byte-neutral opt-in (Species.body_plan becomes Option; morphogen: None keeps sample_body_plan).

## approach

Keep one shared, world-scoped MorphogenProgram (a data-defined axis registry, Vec<AxisSpec>, already Principle-11-clean) threaded alongside BodyPlanRegistry, not stored per-species: genesis.rs's LivingWorld and WorldGenesis gain `pub morphogen: Option<MorphogenProgram>` (Clone, sibling to `registry`), and `genesis()`/`WorldGenesis::new()`/`generate()`/`sample_candidate()` each gain a new `morphogen: Option<&MorphogenProgram>` parameter threaded exactly the way `reg: &BodyPlanRegistry` already is. Diversity across species comes from the genome, not from distinct programs: `GeneratorParams` gains a reserved `ploidy: usize` (basis: matches Race::ploidy()'s SexualDiploid=2 default, but is data so a haploid/clonal alien lineage is a legitimate world choice through the same field) and a reserved GaussApprox for the morphogen block's additive draw. `sample_candidate`, when `morphogen` is `Some(program)`, skips `sample_body_plan` and instead draws a per-species RANDOMIZED target-fraction seed for every one of `program.param_count()` morphogen parameters (via `rng.unit_fixed` at a fresh counter block, clear of the existing niche/draws_on counters), then calls the existing `append_morphogen_block` on top of the SAME flat p.loci niche prefix the pool already carries, and sets `pool = GenePool::new(...).with_additive(effects, gauss)`. This closes a central gap the grounding surfaced: appending the morphogen block onto today's all-zero-effect prefix would make every species express to the identical trivial one-segment body (frac()==0 everywhere maps every axis to its `lo` bound and the spawn threshold to zero, so growth halts at the root for every species alike), a randomized per-species additive spine is required for Arc 6 to do anything at all, not an optional refinement. A SPECIES-representative Structure is grown on demand, never cached: a new pure helper `representative_structure(species_id, &pool, program, niche_loci, ploidy, seed)` promotes one genome via `pool.promote(seed, species_stable_id(species_id).0, ploidy)` (the same `StableId` fold precedent as `OccupantId` at genesis.rs:802), builds an INDEX-ALIGNED GeneSet via a new `morphogen_gene_set_with_prefix(niche_loci, program)` in morphogen.rs (a real, previously unflagged gap: `GeneSet::express` indexes genes by Vec position, so a raw `morphogen_gene_set(program)` starting at index 0 would misalign against a genome whose morphogen loci sit AFTER the niche loci; a prefix of `niche_loci` no-channel placeholder GeneDefs fixes the alignment, mirroring how the founder path prefixes `race.genes.genes.clone()` before appending its own blocks), expresses via the existing `express_program`, and grows via the existing `grow`. Regrowing on demand rather than caching a Structure on every Species is deliberate: every genesis.rs consumer of a species' anatomy (`consumer_bodies`, `producer_compositions`, `place_species`, `describe`) runs once at world genesis or on rare inspector reads, never per tick, so the bounded, deterministic `grow` recursion is cheap there, and this avoids storing a redundant, identical-shaped segment graph on every one of potentially hundreds of species for the life of the world; it also matches the codebase's own established purity claim for the founder path ("growth is a pure function... reproduces identically whether computed at dawn or on a two-tier reload"). A new `Structure::whole_body_composition_vector()` in morphogen.rs closes the one real missing primitive: it unions every axis key present in any segment's `material` map and takes the development-weighted mean per axis, generalizing `composition_sum`/`composition_mean` from one axis at a time to the whole vector exactly as `physiology::whole_body_composition_vector` itself generalizes `whole_body_energy_density`/`body_density` (its own doc comment says so). `digest().body_mass` already supplies body mass with no new code.

## species_struct_change

Change `Species.body_plan` from `pub body_plan: BodyPlan` to `pub body_plan: Option<BodyPlan>` (`Some` = today's catalog tier, byte-identical at every consumer; `None` = grown tier, whose representative Structure is regrown on demand from `species.pool` plus the world's shared `morphogen: Option<MorphogenProgram>`, never cached on Species). No enum and no second field: this reuses the SAME optionality convention `Race` already established (`race.body: Option<BodyPlan>` / `race.morphogen: Option<MorphogenProgram>`), adapted to the fact that Species, unlike Race, has no separate walker to hang the grown tier off, so the None arm itself is the discriminator. GenePool on Species needs no new field; when grown, its loci simply extend from `p.loci` niche-bookkeeping loci to `p.loci + program.param_count()`, exactly the shape `epoch.rs`'s `sp.pool.loci()` (read dynamically at epoch.rs:198) and `selection_coefficients` already handle for whatever locus count a pool carries. Ripple: `epoch.rs:264`'s `body_plan: parent.body_plan.clone()` needs no logic change, only the type update: `Some(bp).clone()` reproduces today's inheritance exactly, `None.clone()` is `None`, and a grown daughter's morphology diverges from her parent automatically and for free because `GenePool::found` (already called at speciation, epoch.rs) preserves the additive-effect spine while re-drawing allele FREQUENCIES under a founder-effect draw (genome.rs:1036-1037): divergence emerging from population genetics rather than a hand-copied field, replacing the epoch.rs comment "body-plan drift is a later refinement" with a mechanism. genesis.rs's four production consumers (`consumer_bodies` line 238, `producer_compositions` line 288, `place_species` line 810, `describe` lines 368-390) each need a match on `&species.body_plan`, `Some(bp)` taking today's call, `None` regrowing via `representative_structure` and reading the Structure equivalent; `describe()`'s None arm has no catalog-part-name analog (a grown Structure carries no kind ids by design) and is flagged as an honest, reduced-fidelity adaptation (segment count, depth, per-function-law capability scores) rather than a blocker.

## byte_neutral_path

Yes. Every existing call site (genesis(), WorldGenesis::new(), every test in crates/sim/tests/, every production caller) passes morphogen: None by default (a new parameter, so callers must be touched, but the value flows through unchanged). With morphogen: None, sample_candidate takes the exact unchanged branch: sample_body_plan(...) runs exactly as before, the pool is built exactly as before (GenePool::new with no .with_additive, an all-zero effects spine, reproducing today's behavior), and body_plan: Some(sampled_plan). Every downstream match takes the Some arm unconditionally, so no state_hash, no composition vector, no body_mass, and no describe() text differs from today's pinned outputs. What re-pins is scoped exactly to a NEW world config that explicitly opts in with morphogen: Some(program): that world's species roster, composition vectors, matter-cycle seeding, and occupant body masses become new content (expected, not a regression) with no effect on any existing morphogen: None world's hash. The one non-hash ripple even at None: every Species { body_plan: ... } struct literal in biosphere.rs's test module, epoch.rs, and crates/sim/tests/biosphere_steering.rs (8 sites total, confirmed by grep) needs Some(...) wrapped around its existing value, a source-level compile fix, not a behavior or hash change. GeneratorParams also stays #[derive(Copy)] (only usize/GaussApprox scalar fields added), and GenesisParams is untouched (it stays Copy, confirmed load-bearing at WorldGenesis::new's params: *params), since the non-Copy MorphogenProgram is threaded as a separate argument and stored only on the non-Copy LivingWorld/WorldGenesis, mirroring exactly how registry: BodyPlanRegistry is handled today.

## derive_vs_author

No new value is authored outside the physics floor. MorphogenProgram's axis lists (geometry_axes/material_axes/composition_axes, each a floor-range AxisSpec) are already the physics floor itself, exactly as the module doc states ("a canonical program reads them from the floor registry"); dev_default() is explicitly labelled a non-canon development fixture with reserved-with-basis ranges, the same status every other floor constant in this codebase carries pre-Arc-6, so reusing it for the biosphere generator introduces no new authoring, only a new consumer of an existing, already-flagged fixture. The two genuinely new reserved values this design needs (ploidy: usize, the morphogen block's GaussApprox) are each surfaced with an explicit basis, never invented, matching the audit-log discipline. Every other new quantity in the design is DERIVED: the per-species target-fraction seeds are RNG draws (data, not authored constants, exactly parallel to the existing niche optimum/breadth draws already in sample_candidate); the grown Structure, its composition vector, and its body mass are pure reads over grown physics with no organ registry and no kind id; the daughter's divergent morphology at speciation is a read of population-genetics machinery (GenePool::found) that already exists and needed zero new code. The MorphogenProgram dev fixture used as a temporary stand-in for owner canon is legitimate under Prime Directive 6 for exactly the same reason it was already legitimate for the founder path: it sits at the physics-floor tier the directive explicitly permits authoring at, and it is labelled, not silently treated as settled.

## alien_feasibility

The AxisSpec list on MorphogenProgram is a bare (axis: String, lo: Fixed, hi: Fixed) with no chemistry, kingdom, or Earth assumption baked in, so a genuinely alien species is a data row with zero code change: a silicon-lattice, non-metabolizing body is reachable today by a species whose randomized additive spine happens to land it near hi on mat.indentation_hardness (up to 3000) and near lo (0) on bio.water_fraction and bio.energy_density; the zero-for-absent convention (morphogen.rs:209-217) means a body that expresses no water fraction and no metabolic energy density is not excluded or defaulted to a Terran baseline, it simply reads zero on those axes and derives its physiology (or lack of one) from that read, exactly the same mechanism that already lets a catalog organ-less body read a zero reserve. A mana-fed or photosynthetic-only species needs no rewrite either: since AxisSpec.axis is an arbitrary floor-axis string, a world whose physics floor defines a mana or light-flux axis (e.g. "mana.flux_capacity") need only append it to the SAME composition_axes list this design already threads, and the biosphere generator's randomized-seed draw (build step 5) covers it automatically with no biosphere.rs code change at all, since it iterates 0..program.param_count() generically. ploidy being reserved data on GeneratorParams rather than hardcoded to 2 means a haploid or clonal alien lineage is equally a legitimate world choice through the same field, not a special case. The one place this design falls short of full alien admission is describe()'s grown-tier arm, which reads segment count and per-function-law capability scores rather than any named organ, so a grown alien's superfine inspector text is coarser than a catalog species' text: an honest, flagged limit, not a silent Terran default.

## risks

The central correctness risk the grounding surfaced and this design fixes is the flat-additive-spine collapse: naively appending the morphogen block onto the species pool's existing all-zero effects (as today's sample_candidate builds it) would make every species express to the identical trivial one-segment body, the opposite of Principle 8 and worse than today's arbitrary-but-varied catalog sample; the randomized per-species seed draw (build step 5) is required, not optional. The second, previously unflagged risk is the GeneSet index-alignment gap: GeneSet::express indexes by Vec position, so using the bare morphogen_gene_set(program) (indices 0..param_count) against a genome whose morphogen loci sit AFTER the niche loci would silently read the WRONG locus for every parameter (a silent-corruption bug, not a crash), caught only by tracing express's enumeration contract; morphogen_gene_set_with_prefix closes it. A design risk carried forward rather than fixed: epoch.rs's selection_coefficients applies one uniform per-species fitness scalar across every locus including morphogen loci, so selection shapes body shape only as a side effect of a generic regional-fit scalar, never because a specific grown Structure's own capability or viability was read; this is an honest limit to surface, not a blocker, and is a natural follow-on research item (a fitness term reading Structure::whole_body_viability per candidate). Performance risk is low (regrowth is bounded and runs only at one-time genesis / rare inspector reads, never per tick, confirmed by grepping all production callers of consumer_bodies/producer_compositions/describe). The describe() None-arm loses named-part fidelity for a grown species by design (a grown Structure carries no catalog kind ids); this is an intentional, flagged trade rather than an oversight.

## Build steps

1. morphogen.rs: add Structure::whole_body_composition_vector(&self) -> BTreeMap<String, Fixed>, unioning every axis key present across all segments' material maps and taking the development-weighted mean per axis (uniform 1/MASS_REFERENCE_SEGMENTS-consistent weight, matching composition_mean's discipline), generalizing composition_sum/composition_mean exactly as physiology::whole_body_composition_vector generalizes whole_body_energy_density/body_density.

2. morphogen.rs: add morphogen_gene_set_with_prefix(prefix_loci: usize, program: &MorphogenProgram) -> GeneSet, prepending prefix_loci no-channel placeholder GeneDefs (empty effects, so GeneSet::express's per-locus channel_weight lookup returns None and contributes zero) before the program's own per-parameter GeneDefs, so a caller whose pool carries loci ahead of the morphogen block gets index-aligned expression through the existing, unmodified express_program/GeneSet::express (which index by Vec position, not GeneId).

3. biosphere.rs GeneratorParams: add pub ploidy: usize (RESERVED, basis: match Race::ploidy()'s SexualDiploid=2 default for a grounded fixture; an equally legitimate haploid/clonal alien choice through the same field) and a reserved GaussApprox constant/field for the morphogen block's additive draw (RESERVED, basis: cross-tier consistency with whatever the founder morphogen path settles on, itself currently only a test fixture).

4. biosphere.rs Species: change pub body_plan: BodyPlan to pub body_plan: Option<BodyPlan>.

5. biosphere.rs generate/sample_candidate: add a morphogen: Option<&MorphogenProgram> parameter threaded alongside reg: &BodyPlanRegistry. When None, take today's unchanged path (body_plan: Some(sample_body_plan(...)), flat-zero pool). When Some(program): skip sample_body_plan; draw a per-species randomized target-fraction seed for each of program.param_count() parameters via rng.unit_fixed at a fresh counter block clear of the existing niche/draws_on counters (R-RNG-COORD discipline); call append_morphogen_block on top of the existing p.loci flat niche prefix; set pool = GenePool::new(...).with_additive(effects, gauss); set body_plan: None.

6. biosphere.rs (or a small shared module): add species_stable_id(id: SpeciesId) -> StableId (trivial deterministic wrap, precedented by genesis.rs:802) and representative_structure(species_id, &pool, program, niche_loci, ploidy, seed) -> Structure, composing pool.promote + morphogen_gene_set_with_prefix + express_program + grow. Pure, deterministic, uncached.

7. genesis.rs: add pub morphogen: Option<MorphogenProgram> to LivingWorld and to WorldGenesis (Clone, not Copy, sibling to registry); thread a new morphogen: Option<&MorphogenProgram> argument through genesis() and WorldGenesis::new(), passed into every generate(...) call alongside &registry.

8. genesis.rs consumer_bodies (line ~238) and producer_compositions (line ~288): fork on &species.body_plan; Some(bp) => whole_body_composition_vector(bp, &self.registry) unchanged; None => representative_structure(...).whole_body_composition_vector(), using self.morphogen.as_ref().expect(a grown species requires a world morphogen program) (fail-loud on the invariant violation, never a silent default).

9. genesis.rs place_species (line ~810): fork body_mass the same way, None arm reading representative_structure(...).digest().body_mass.

10. genesis.rs describe() (lines ~368-390): fork on sp.body_plan; Some keeps the unchanged catalog-name path; None returns a physics-derived description (segment count, depth, per-function-law max_capability scores) instead of named parts, flagged as reduced-fidelity, not blocked.

11. epoch.rs:264: update the type only (parent.body_plan.clone() stays correct code under Option<BodyPlan>); no new logic needed since GenePool::found already preserves the additive spine while re-drawing frequencies at speciation, so a grown daughter's morphology diverges from her parent for free.

12. Fix the 8 Species { body_plan: ... } struct literals across biosphere.rs's test module, epoch.rs, and crates/sim/tests/biosphere_steering.rs to wrap the existing value in Some(...).

13. Add tests: a Some(MorphogenProgram::dev_default()) world where species differ in grown morphology from their first draw (proving the randomized-spine fix avoids the flat-spine collapse), a daughter's Structure diverges from its parent post-speciation, a grown-tier analog of generated_species_have_distinct_deterministic_anatomy, and a state_hash-equality regression proving the None default path is unchanged bit-for-bit against the pre-change baseline.

14. Run cargo fmt --all, then cargo test -p civsim-sim for biosphere/genesis/epoch, plus a full dev-fixture world determinism run under morphogen: None to confirm zero drift.


## Map

GROUNDED MAP OF THE MORPHOGEN-TO-BIOSPHERE GAP, with file:line citations.

=== 1. The founder Structure growth pipeline, end to end (proven, off-biosphere) ===

A sentient race opts in with `Race::with_morphogen(program)` (crates/sim/src/race.rs:204-207), which sets `race.morphogen: Option<MorphogenProgram>` (race.rs:123), sibling to the catalog `race.body: Option<BodyPlan>` (race.rs:112). Both stay `None` until declared (fail-quiet-until-declared), so an existing catalog race is untouched.

Founder assembly (crates/sim/src/worldbuild.rs:562-590, inside `assemble_dawn_embodiment`): for each founder id in a band, `let structure = match (&race.morphogen, world.genome_of(id)) { (Some(program), Some(genome)) => { let params = express_program(program, &race.genes, genome); Some(grow(program, &params, seed, id)) } _ => None }` (worldbuild.rs:565-571). The metabolic body/reserves are then sourced from whichever tier is present: a catalog `race.body` keeps `Homeostasis::new(&genesis.homeostatic, plan, &genesis.organs)` (homeostasis.rs:279-306); a fully grown race with no catalog body instead uses `s.digest()` for the LOD-0 `BodyPlan` and `Homeostasis::from_structure(&genesis.homeostatic, s)` (homeostasis.rs:317-328) (worldbuild.rs:578-590). `Walker::new(...)` builds the walker (locomotion.rs:640-675), then `walker.with_structure(s)` attaches the full segment graph (worldbuild.rs:626-628; builder at locomotion.rs:682-685).

`express_program(program, genes, genome)` (morphogen.rs:445-455) is a pure sum-over-loci read, one `Fixed` per morphogen-parameter id, through `GeneSet::express(genome, Channel::Morphogen(MorphogenParamId(k)), ZERO)` (genome.rs:404-426), exactly the same mechanism `Channel::Controller` weights use. The loci feeding those channels are appended to a founder's gene set and pool by `append_morphogen_block` (genome.rs:499-530), the direct sibling of `append_controller_block` (genome.rs:458-489): seeded parameters get a deterministic-homozygote frequency of ONE and a dosage-normalised `target/ploidy` additive effect; unseeded ones get a balanced 1/2 frequency and additive effect ZERO (expresses to zero until mutation drifts it). This wiring is exercised only in tests today (crates/sim/tests/world_build.rs:1425-1471, `morphogen_race`/`morphogen_peoples`); no production race in the shipped world-build actually calls `with_morphogen` yet.

`grow(program, params, seed, id: StableId)` (morphogen.rs:490-552) recurses under a hard depth/segment/branch cap (`max_depth`, `max_segments`, `max_branch`, all RESERVED-with-basis on `MorphogenProgram`, morphogen.rs:95-104), counter-keyed on `Phase::MORPHOGEN` via `DrawKey::pair(id.0, locus, gen, Phase::MORPHOGEN)` (morphogen.rs:533, 573), so growth is a pure, deterministic function of `(program, params, seed, id)` and folds into `state_hash` through `Structure::hash_into` (morphogen.rs:420-440). Each `Segment` carries a `geometry: BTreeMap<String, Fixed>` (mech.* axes) and `material: BTreeMap<String, Fixed>` (mat.*/opt.* axes plus the tissue-composition axes: bio.convective_surface, mat.fracture_strength, therm.specific_heat, bio.energy_density, bio.water_fraction, morphogen.rs:175-181), read with a zero-for-absent convention (morphogen.rs:209-217), identical in shape to a catalog `KindDef`'s material map but with no kind id at all.

The newborn path (crates/sim/src/runner.rs:5343-5419, `embody_newborn`) mirrors this exactly: `let structure = match (&race.morphogen, world.genome_of(id)) { (Some(program), Some(genome)) => { let params = express_program(program, &race.genes, genome); Some(grow(program, &params, emb.seed, id)) } _ => None }` (runner.rs:5358-5364), same digest/from_structure fork (runner.rs:5370-5382), same `walker.with_structure(s)` (runner.rs:5417-5419). Growth is a pure function of `(program, genome, seed, id)`, so it reproduces identically whether computed at dawn or on a two-tier reload where the walker is regrown from the re-minted genome (walkers are never serialized).

=== 2. What a Structure supplies that a BodyPlan does not (composition, mass, capability) ===

`Structure` (morphogen.rs:225-441) supplies, all as pure reads over the grown segment graph, with NO organs registry and NO kind id:

- Composition: `composition_sum(axis_id)` (morphogen.rs:313-323), the extensive Sigma over segments of `(1/MASS_REFERENCE_SEGMENTS) * segment.material[axis]`, used for reserve capacities, exchange surface, muscle force; `composition_mean(axis_id)` (morphogen.rs:329-340), the intensive mean, used for specific heat/energy density; `backed_capacity(axis_id)` (morphogen.rs:346-348, delegates to composition_sum) and `whole_body_energy_density()` (morphogen.rs:354-356, delegates to composition_mean on "bio.energy_density"). These are the SAME two-axis-at-a-time primitives `whole_body_energy_density`/`body_density` use over a catalog BodyPlan (physiology.rs:255-273), but read directly off segments instead of `plan.organs` against `BodyPlanRegistry::organ_composition`.
- Mass: `digest()` (morphogen.rs:391-418) derives `body_mass` from `segment_count / MASS_REFERENCE_SEGMENTS` (clamped to [0,1], MASS_REFERENCE_SEGMENTS=16, morphogen.rs:623), the LOD-0 aggregate `BodyPlan`, leaving weapons/senses/locomotion/organs empty by design ("a grown body's FUNCTION is read from the structure directly ... never from the digest's part-lists").
- Capability: `segment_capabilities(i, fns, refs, caps)` (morphogen.rs:249-264) reads one segment's `CapabilityVector` through `civsim_compose::derive_capabilities` on its own geometry/material, blind to any id; `max_capability(law, ...)` (morphogen.rs:269-281) is the greatest capability any segment reads on one function law; `whole_body_viability(...)` (morphogen.rs:291-302) is the greatest capability ANY segment reads on ANY function law, the run-tier "is this body functionally coherent at all" read; `best_locomotor_stride(...)` (morphogen.rs:362-380) is the strongest LOCOMOTE segment's capability and its `mech.arm_length`.

These feed the run at: `AffordanceRegistry::afforded_structure` vs `afforded` (runner.rs:1314-1326, `being_afforded`); `locomotion_speed_structure` vs `locomotion_speed` (runner.rs:871-877, exertion-drain velocity); `being_muscle_force` (runner.rs:804-812, `composition_sum(MUSCLE_STRENGTH)` scaled by body mass, vs catalog `whole_body_muscle_force`); the metabolic derived-drain block (runner.rs:844-857, energy density/surface/force from the structure vs catalog organs); `walker_exchange_rate` (runner.rs:907-929, `composition_sum(CONVECTIVE_SURFACE)`/`composition_mean(TISSUE_SPECIFIC_HEAT)` vs catalog); and the viability cull (runner.rs:4900-4920, `whole_body_viability` sets the INTEGRITY reserve each tick, so an incoherent grown body dies through the ordinary reserve-floor cull, no morphology-inspecting predicate).

IMPORTANT SCOPE NOTE: every one of those run-path reads is keyed on `Walker` / `Embodiment` (locomotion.rs:503-635), i.e. an individually embodied, ticking being. Biosphere creatures are NOT walkers (see section 3). None of the affordance/locomotion/exchange-rate/viability-cull machinery is reachable by a biosphere species as the architecture stands; only the aggregate composition/mass reads (`composition_sum`/`composition_mean`/`digest`) are relevant to what Arc 6 actually needs to replace.

=== 3. Species.body_plan today: how it is sampled, and its granularity ===

`Species` (biosphere.rs:153-164) carries ONE `body_plan: BodyPlan` and ONE `pool: GenePool` for the whole species (a population aggregate), not a per-individual body or genome. `sample_candidate` (biosphere.rs:454-510) builds the pool at biosphere.rs:491-496: `GenePool::new(SchemeId(0), p.pool_size, vec![Fixed::from_ratio(1,2); p.loci])`, no `.with_additive(...)` call, so `effects` is implicitly all-ZERO (GenePool::new's default, genome.rs:846-855) and the pool never expresses to any phenotype channel today (it exists purely for the epoch's Hardy-Weinberg drift/selection/speciation bookkeeping in epoch.rs, never fed through `GeneSet::express`). Immediately after, `sample_body_plan(rng, layer, p.rooted_prior(layer), reg, profile, 200)` (biosphere.rs:501, calling anatomy.rs:689-761) draws the CATALOG BodyPlan by picking distinct kind ids from `BodyPlanRegistry` lists (weapons, coverings, senses, locomotion, organs, each via `pick`/`pick_organs`, anatomy.rs:625-677) keyed on plain RNG counters, no genome or pool involvement at all. This is the sampled BodyPlan Arc 6 must replace.

Biosphere occupants are placed by `genesis.rs`'s `place_species` (genesis.rs:780-815): one `OccupantInfo { species: SpeciesId, layer, region, body_mass: sp.body_plan.body_mass }` (genesis.rs:804-812) per accepted species per region, scattered onto the map by a deterministic hash. There is no per-individual creature identity, no promoted individual `Genome`, and no `world.genomes` entry for any biosphere organism (that store is populated only by `world.seed_dawn_populations` for sentient-race founders, world.rs:1335-1362, via `race.pool.promote(self.seed, id.0, race.ploidy())` at world.rs:1353). So growing "a species' Structure" is naturally a species-level operation (one Structure per species, matching the existing one-BodyPlan-per-species granularity), not a per-organism one; no architecture change to occupant granularity is implied.

=== 4. What the biosphere needs to grow a species Structure from its pool genome ===

(a) A shared, data-defined `MorphogenProgram` threaded into `biosphere::generate`/`sample_candidate` (biosphere.rs:361-368, 454-462) and ultimately `genesis.rs`'s `GenesisParams`/`genesis()` (genesis.rs:53-114, 433-445), sibling to how `reg: &BodyPlanRegistry` is threaded today. `morphogen_gene_set(program)` (morphogen.rs:460-472) is a pure function of the program alone (locus k feeds `Channel::Morphogen(MorphogenParamId(k))` uniformly), so one canonical program/GeneSet can serve every species; no per-species GeneSet storage is needed.

(b) The species pool must carry the morphogen loci, appended via `append_morphogen_block` (genome.rs:499-530) on top of the existing `p.loci` niche/bookkeeping loci (loci count becomes `p.loci + program.param_count()`), exactly mirroring the Race recipe (`append_controller_block` then `append_morphogen_block`, proven in crates/sim/tests/world_build.rs:1436-1470). CENTRAL GAP: the current species pool construction (biosphere.rs:491-496) has a flat, all-zero additive spine. If the morphogen block were appended with that same flat spine (freq 1/2, effect 0 at every locus, no seeds), `express_program` would return the ZERO vector for every species regardless of drawn allele state (GeneSet::express's genotypic term is `additive + dominance-if-heterozygous`, and `additive` is built from `per_allele_additive[locus]` in `GenePool::promote`, genome.rs:1083-1104, which is forced to `Fixed::ZERO` whenever `alpha == Fixed::ZERO`, genome.rs:1089-1091). `frac()==0` for every parameter maps every geometry/material axis to its `lo` bound (morphogen.rs:636-638) and the spawn threshold to zero, so `spawn >= spawn_threshold` is true for the very first draw and no children ever grow (morphogen.rs:537-539): every species in the world would grow the IDENTICAL trivial one-segment body. That is a worse failure than today's arbitrary catalog sample: it collapses ALL species to one uniform morphology, the opposite of Principle 8. The generator therefore needs an analogous per-species randomized quantitative-spine draw for the morphogen block's additive effects (paralleling the existing per-axis `optimum`/`breadth` draws at biosphere.rs:463-472, which already use `rng.unit_fixed` at counter offsets past the niche counters), so a species' morphology starts genuinely differentiated and then diverges further by drift/selection, rather than defaulting to a shared flat spine.

(c) Ploidy: `GenePool::promote(seed, individual_id, ploidy)` (genome.rs:1079-1131) takes ploidy as a caller-supplied `usize`; `Species` carries no `GeneticScheme` (unlike `Race`, which derives ploidy from `race.scheme.reproduction` via `Race::ploidy()`, race.rs:218-223). This is a genuinely new datum the generator needs (RESERVED with basis: match the diploid default sentient races use, or add a `ploidy` field to `GeneratorParams`, biosphere.rs:229-259, since an alien/haploid species is a legitimate generative point too).

(d) One representative genome per species must be promoted from the (now morphogen-extended) pool: `species.pool.promote(seed, some_u64_from(species_id), ploidy)`, precedented exactly by world.rs:1353. `grow` needs a `StableId`, not a `SpeciesId(u32)` (lineage.rs:39); this is a trivial deterministic wrap, precedented by genesis.rs:802's existing `StableId((region_id << 20) ^ id.0 as u64)` fold for `OccupantId`.

(e) `digest()` (morphogen.rs:391-418) directly supplies `body_mass` for the two sites that need it: `genesis.rs:810` (`OccupantInfo.body_mass`) and the `consumer_bodies` volume calculation (genesis.rs:243-246 via `info.body_mass`). Drop-in once a per-species digest exists.

(f) Composition-vector gap (a real missing piece, not yet in morphogen.rs): `genesis.rs:238` and `genesis.rs:288` both call `whole_body_composition_vector(&species.body_plan, &self.registry)` (physiology.rs:290-...), which unions EVERY floor axis any catalog organ/covering/weapon declares (via `BodyPlanRegistry::organ_composition` and each `KindDef.material`) into one development-weighted-mean vector, the SAME fold a corpse deposits and a grazer/predator reads. `Structure` has no equivalent whole-vector read today, only the single-axis `composition_sum`/`composition_mean`. A new function generalizing those two exactly as `whole_body_composition_vector` itself generalizes `whole_body_energy_density`/`body_density` (its own doc comment says so explicitly, physiology.rs:275-289) is required in morphogen.rs: union every axis key present across all segments' `material` maps, then the development-weighted mean per axis (uniform per-segment weight `1/MASS_REFERENCE_SEGMENTS`-consistent, matching `composition_mean`'s discipline), no registry, no kind id at all, mechanically small but currently absent.

(g) `describe()` (genesis.rs:311-392) reads catalog part NAMES for the superfine inspector (`bp.weapons`/`bp.senses`/`bp.covering` looked up via `BodyPlanRegistry::name`, genesis.rs:370-390). A grown Structure carries no catalog kind ids whatsoever by design (morphogen.rs's own module doc: "no catalog kind id and no organs registry"), so there is no direct analog; a Structure-grown species needs either a different, physics-derived description (segment counts, per-law capability scores via `max_capability`) or this consumer stays gated to catalog-body species. Flagged as an honest adaptation gap, not a blocker.

(h) `epoch.rs:259-268` (`crate::biosphere::Species { ..., body_plan: parent.body_plan.clone(), ... }`, the daughter-inherits-parent's-anatomy line, with the comment "body-plan drift is a later refinement"). Once Species stores a morphogen-extended pool instead of a cached `body_plan`, this line disappears: `GenePool::found` (genome.rs:1002-1038), which already builds every daughter pool at speciation (epoch.rs:238-241), resamples per-locus FREQUENCIES under a founder-effect draw but explicitly PRESERVES the additive spine (`GenePool::new(self.scheme, recovery_size, freqs).with_additive(self.effects.clone(), self.gauss)`, genome.rs:1036-1037). So once (b) supplies a real per-locus effect vector, a daughter's grown Structure diverges from its parent's automatically, for free, purely because its promoted genome's allele STATES differ (the frequencies moved) while the additive MEANS travel unchanged: an actual instance of Principle 8 (order/divergence emerging from population genetics) replacing a hand-copied field.

(i) Selection-coupling caveat, pre-existing and now newly load-bearing: `epoch.rs:135-146` (`selection_coefficients`) computes ONE uniform coefficient from regional niche suitability and applies it to every locus (`loci = sp.pool.loci()`, read dynamically at epoch.rs:198, so it already covers however many loci the pool carries, including a future morphogen block automatically). It carries no link to whether the GROWN Structure's own derived viability/capability actually fits anything; it is a blanket per-species push toward fixation, well-fit pushing toward state ONE, poorly-fit against it, identical for the niche loci and any morphogen loci riding the same pool. This is not a defect Arc 6 introduces, but it becomes materially relevant the moment morphogen loci carry real phenotypic effect: selection would shape body shape only as a side effect of a generic regional-fit scalar, not because a particular grown morphology is more capable. Worth surfacing as an honest limit alongside the wiring, not a blocker to it.
