# Determinism and Reproducibility Audit: Consolidated Report

## Executive summary

The Emergent Civilization Simulator rests its whole observer-independence and shareability promise on one rule: one seed produces one world, bit for bit, on any machine and at any thread count. This audit ran a red-then-green process over eleven dimensions, from the fixed-point numeric core through the GPU quarantine, scheduling, event sourcing, LOD tier transitions, and the higher subsystems (belief, genetics, language, ecology, technology). The verdict is encouraging in its foundations and pointed in its defects.

The principles are coherent and several of them are defended by sound mechanisms in the built bedrock. The typed canonical boundary makes a float in canonical state a compile error. Fixed addition is exact and associative in range, so a partitioned sum is order independent. The counter RNG's bounded draw uses a portable 128-bit multiply-high, and its SplitMix64 finalizer gives distinct streams for distinct ids. The conservation registry measures gross totals in i128 so the totals it tracks are conserved exactly.

Against that, the audit confirmed ten findings as real. The single most severe is implemented: a promoted entity's entire random stream is keyed on a StableId minted in allocation order, so the order in which quiet regions promote, which view-time elaboration can perturb, changes canonical ids and the state hash. That is the exact collision the design's tier-consistency ruling was written to forbid. Three more implemented defects are clear and fixable now: a signed-range RNG helper that overflows, a float-to-canonical quantizer that rounds at the wrong stage, and the default Fixed accumulation operators that panic on overflow rather than carrying the per-quantity discipline the design itself mandates. Referential integrity across pool merge and split is the other recurring implemented weakness, because the integrity check tests only that a key exists, not that the location it names is still live.

The larger surface is design level. Many invariants are asserted without being structurally guaranteed: HashMap-backed canonical containers with no enforced sorted walk, unspecified tie-breaks and reduction orders for non-associative combines, GPU cross-backend bit-identity for multiply, divide, transcendentals, worldgen noise, and the flow-field eikonal that is claimed but never pinned to a single integer algorithm, save schema versioning that the load-bearing snapshot struct omits, and a stale camera-drives-promotion passage that contradicts the governing ruling. None of these is a live divergence in shipped code, because the subsystems that would trip them are unbuilt, but each is a foot-gun the first implementer can follow straight into a broken contract.

## Confirmed vulnerabilities

The findings below carry a "real" verdict from the green pass. The first set is against the implemented bedrock and is fixable now, which is why each is paired with a reproduction test in the recommendations.

**C-01, critical, observer independence (implemented).** A promoted entity's stream is `Rng::for_entity(S, id, phase)` where `id` comes from `Registry::mint`, a plain monotonic counter (id.rs:74-81) called inside `promote` (lod.rs:124-129). Two runs that promote two quiet regions in swapped order mint swapped ids, so each member gets a different key, a different draw stream, and a different state hash. If promotion order can be influenced by view-time elaboration touching regions in a different order, watching moves the timeline. The design (Part 54 lines 2920, 2928) requires the key to be a pure function of master seed, region, tick, event, and slot, with ids derived deterministically; the bedrock keys off an allocation-order id instead.

**C-02, high, determinism (implemented).** `range_i32` computes `lo + self.range_u32(counter, span) as i32` (rng.rs:89-93). With `lo = i32::MIN, hi = i32::MAX` the span is `u32::MAX`, a draw can produce `0xFFFFFFFE` which casts to `-2`, and `i32::MIN + (-2)` overflows. Under the repo's overflow checks this panics; otherwise it returns a value outside `[lo, hi)`. The fix widens the addend to i64 before the cast.

**C-03, medium, determinism (implemented).** `quantize_unit` applies `round_ties_even` to the count of fractional units, then truncates the final integer division (canonical.rs:85-91), so the only sanctioned float-to-canonical crossing is neither exact nor round-half-to-even as its own doc claims. `quantize_unit(1.0/3.0, 1000)` yields bits 1430224109 against the nearest Fixed's 1431655765. The correct form rounds the full Q32.32 product to even.

**C-04, high, conservation (implemented and design).** The default `Add`, `Sub`, `AddAssign`, and `Sum` for Fixed use raw `+`/`-` (fixed.rs:178-238), and the tier-transition wealth moves in lod.rs use the same operators. `saturating_add` and `checked_add` exist but are wired into nothing on the accumulation path. With overflow checks on in both profiles (Cargo.toml:20), a century-scale stock that crosses i64 range panics, a hard determinism failure, and the i128 audit measure would not catch a wrap if one occurred. Part 55 line 2976 requires a defined per-quantity saturate-or-wrap choice; the bedrock provides neither.

**C-05, high, determinism (implemented and design).** Because checked addition is not associative as a partial function, a parallel Fixed sum is panic-versus-success depending on the chunk boundary. The multiset `[i64::MAX, 10, -10]` (in bits) overflows on a sequential fold but completes under a `{[MAX]},{[10,-10]}` partition. The harness partitions by `len.div_ceil(threads)` (determinism.rs:50,126), so which outcome occurs is a function of thread count. Part 3.3 line 341 and the fixed.rs docs assert the parallel sum is unconditionally safe; it is safe only while every partial stays in range.

**C-06, high, referential integrity (implemented).** `demote` records `Pooled{pool, slot: count-1}` (lod.rs:144-150), but `merge_pools` removes the absorbed pool without re-homing any member's location, and `split_pool` never re-indexes slots. `resolves` checks only `contains_key` (id.rs:95-97), so `referential_integrity_ok` reports valid while a location names a pool that no longer exists. The demote slot is also non-unique, because every demote into a pool assigns `count-1`.

**C-07, medium, conservation (implemented and design).** `promote` and `split_pool` subtract a caller-supplied share with only a count guard, never a wealth guard, and never invoke the conservation audit. A share exceeding pool wealth drives the pool negative while the gross total stays balanced, so `check_against` passes. The exact integer partition with lowest-id remainder that Part 54 line 2920 mandates is implemented nowhere, so re-promotion cannot reproduce the same individual.

**C-09, medium, conservation (implemented).** `Event::referenced` chains actors then subjects with no dedup, and `append` pushes the EventId per occurrence (event.rs:87-92, 117-118), so a self-directed event is indexed twice and `history_of` returns it twice. Any multiplicity-based fold over an entity's history over-counts. Deterministic, so the determinism harness passes, but a count-based conserved projection leaks.

**C-08, high, observer independence (design).** Part 11.2 line 1128 and Part 14.5 line 1272 still instruct camera-driven canonical promotion, which contradicts the governing Part 54 ruling (significance, not the camera) and Principle 10. The invariant itself is defended by Part 54 and the Stage 2 acceptance test, but the un-amended prose is a real defect that an implementer building Part 11 in isolation would follow into an observer-dependent engine, and the single promote path carries no structural canonical-versus-elaboration flag.

**C-10, high, referential integrity (design and implemented constraint).** The implemented invariant that EventId is the dense storage index (event.rs:134-138) cannot coexist with Part 7.3 line 753 ("the originals are truncated") and Part 11.3 / Part 54 ("never deleted", "stays append-only"). Truncating demoted-entity events shifts every later index, so stored EventIds in `by_entity` and in cross-store edges then point at the wrong event or out of range. One of the three statements must give.

## Gaps and underspecifications

These carry an "underspecified" verdict: the design does not yet guarantee the invariant, and no shipped code violates it, so each is a candidate research flag or design-hardening note.

**G-01, R-GPU-CANON-PIN.** The cross-vendor bit-identity argument is stated only for integer addition. Multiply and divide need an i128 intermediate and a narrowing wrap that WGSL, SPIR-V, Metal, and DX12 must emulate, and the spec pins neither the emulation nor the rounding direction. The named transcendentals have no precision or termination rule, Nei distance needs a logarithm the Part 3.4 whitelist excludes, worldgen noise and the flow-field eikonal are classically float and never declared canonical, and the launcher selects DX12, GL, and software adapters that lie outside the proven CUDA, Vulkan, Metal, CPU set.

**G-02, R-CANON-WALK.** Canonical identity, location, provenance, and economy state live in std HashMaps with no ordered iterator and no canonical-walk helper, while Part 3.5 line 380 mandates a StableId-sorted walk and the hash primitive is order sensitive. PoolId and EventId lack a derived Ord, and Vecs reordered by `swap_remove` are likewise order underspecified. No current path walks these order-sensitively, but the path of least resistance for any future save, hash, gossip selection, market clear, or unit fold is HashMap order.

**G-03, R-CMD-ORDER.** `command_sort_key` is referenced but never defined and is described as StableId-based, yet Promote and EmitEvent carry no StableId, so a stable sort breaks ties by per-worker concatenation order, which moves with thread count. Spawn pre-mints its id in the parallel ActionStage from a mutable monotonic registry, and EmitEvent is both a Command and written only in the single-threaded Events phase.

**G-04, R-RNG-COORD.** `for_entity` keys only `(seed, id, phase)` with no tick, so a per-tick draw written naively is frozen forever. There is no counter namespacing for two draw-sites in one stream, no shared audited multi-coordinate fold (every subsystem hand-hashes through `from_key`), no phase-number registry, and `range_i32` panics on a reachable empty range while `range_u32` defines it.

**G-05, R-PROJ-REGISTER.** Only population and wealth are registered as conserved projections, though Part 54 says each two-tier subsystem registers what it conserves. Belief mass is folded as a mean, which cannot conserve a total and truncates; genetics has no projection for allele-copy mass or additive variance and rounds promotion and demotion differently; vegetation biomass and soil fertility cross tiers with no projection or partition rule; concept-snap collapses facets with no defined merged strength.

**G-06, R-REDUCE-ORDER.** Several non-associative canonical combines have no pinned order: the gossip sub-batch conflict resolution, `weighted_pick` over a HashMap-sourced candidate list, the migration-blend renormalization, the technology product-of-efficiencies aggregation over SmallVec children, coupled-stock stepping, and the unified graph traversal. The deterministic scheduler and the event-driven wake order are explicitly deferred in Part 57.

**G-07, R-SAVE-SCHEMA.** WorldSnapshot carries no schema version and no migration hook, while rkyv zero-copy ties decoding to an exact layout, so a save becomes unloadable across engine versions. `next_id` is exposed for snapshot but has no restore path or high-water-mark rule, so a reload that derives it from the live-entity count after compaction re-issues a StableId that surviving references alias, and `resolves` cannot detect the alias. On-disk endianness and layout versioning are unpinned.

**G-08, R-HARNESS-COVER.** The binding harness exercises only an order-independent per-entity accumulation, so it cannot catch a command-ordering, gossip-partition, tier-transition, or HashMap-iteration regression. The invariants test runs one fixed sequence on one thread with no state hash, no thread sweep, and no replay compare, which are the three things Part 3.5 says make a divergence catchable.

**G-09, R-UNITS-PIN.** The Part 55 unit system declares rules but defines none: no per-quantity scale table, no canonical conversion direction, no per-quantity overflow policy, lossy emic round-trips for non-power-of-two scales, and a HashMap unit store. The crate does not exist.

**G-10, R-LANG-DET.** ConceptId is content-hashed rather than minted, so it has no resolves contract and a non-total sort key on collision; sound-change inter-rule order and simultaneous-versus-sequential semantics are unspecified; salience is an unbounded century-accumulating Fixed with no overflow discipline; the shared-concept traversal and distance rounding are unpinned; and the dawn-dynamic trigger is not tied to the significance schedule.

## Notable defended attacks

Several attacks were soundly defended, which is evidence the contract holds where it has been built out.

The bounded RNG draw (D-01) is bit-identical on every machine because `range_u32` uses a portable 128-bit multiply-high; the only residual is a comment that calls it "bounded" when it is the rejection-free fast path. The typed canonical boundary (D-02) makes a float in canonical state a compile error through `Canonical` not being implemented for `f32`/`f64`, plus `NonCanonical` and `CanonicalCell`, which defends the observer-independence-by-construction claim and the float-salience and gloss-backfeed vectors. Exact associative Fixed addition (D-03) makes an in-range reduction order independent regardless of partition or work stealing. The gossip partition (D-04) guarantees write disjointness within a batch, so the data race is closed even though the deferred-write order remains open. The SplitMix64 finalizer (D-05) gives distinct uncorrelated streams for distinct nearby ids, defending the genome-seed-collision claim. The conservation registry (D-06) conserves exactly any gross total it is given, measured in i128. The quantizer architecture (D-07) requires any field that feeds a canonical decision to be a fixed-point kernel with no float variance, so a canonical threshold is not meant to read a non-authoritative float through the quantizer at all.

## Prioritized recommendations

The highest priority is the set of implemented bedrock fixes, because they are actionable now and each can carry a regression test that the rest of the project will lean on.

1. Rederive promoted-entity identity and its RNG key from (master seed, region, tick, event, slot) rather than an allocation-order mint (C-01). High. This is the load-bearing observer-independence fix.
2. Widen the addend in `range_i32` to i64 before the cast (C-02). High, one line.
3. Route every century-scale accumulator and tier-transition wealth move through a defined saturate-or-checked policy, and qualify the "exact" docs as "exact while in range" (C-04, C-05). High.
4. Re-home Pooled locations on merge and split, assign unique stable slots on demote, and make `resolves` and the integrity check validate location liveness (C-06). High.
5. Reimplement `quantize_unit` to round the full Q32.32 product to even (C-03). Medium.
6. Implement the lowest-id-remainder partition, add a non-negativity guard, invoke the audit per transition, and make `check_against` treat a missing baseline as a hard error (C-07). Medium.
7. Dedup `Event::referenced` before indexing `by_entity` (C-09). Medium.
8. Add reproduction tests for C-01 through C-07 and C-09, and extend the determinism harness to cover phases, command buffers, gossip, structural mutations, a thread-count sweep, a StableId-sorted state hash over the two-tier world, and a save/load/replay compare (G-08). High. The harness extension is the prerequisite that makes the other findings catchable the moment those subsystems are built.

The design-level work is best tracked as research flags and hardening notes: R-CANON-WALK and R-CMD-ORDER for sorted iteration and total command ordering (G-02, G-03, with C-10's EventId question), R-GPU-CANON-PIN for the cross-backend numeric and kernel specification (G-01), R-PROJ-REGISTER and R-REDUCE-ORDER for conserved projections and non-associative combine ordering (G-05, G-06), and R-SAVE-SCHEMA for persistence (G-07). Strike or rewrite the stale camera-drives-promotion text and give the promote path a structural canonical-versus-elaboration boundary (C-08), resolve the EventId-index versus truncation contradiction before compaction is built (C-10), and open R-UNITS-PIN and R-LANG-DET for the unbuilt unit and language subsystems (G-09, G-10). A small low-priority hardening item closes the typed-boundary leak by removing or gating `to_f64_lossy` and pinning `to_int`'s truncation direction.
