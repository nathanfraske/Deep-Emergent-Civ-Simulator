# Parked legacy arcs are not canonical

The biology, civilization, dawn, and causal-viewer arcs were built under the
retired methodology. Their code and evidence are preserved under `parked/`, but
they do not define planetary behavior, readiness, or the default developer
path. Root packages may keep narrow compatibility consumers during extraction;
no canonical package may depend on a parked package.

The old viewer is included in that ruling. It constructs and advances planet
state inside an observer loop, so frame cadence and user controls can change the
physical result. It lives at `parked/crates/viewer` as
`civsim-viewer-legacy`. It is useful for archaeology and visual comparison, but
it is not a source of canonical inputs or outcomes.

## Rule for the canonical planet runner

The canonical path is library-owned by `civsim-planet` and entered through its
real binary. It follows these rules:

- Every value is represented by the seven provenance types and one of the four
  ledger tiers. There is no calibration profile or calibration-manifest path.
- The runner accepts only the validated absolute physics floor. A provenance
  tag or citation is accounting, not admission. A value first goes through the
  derivation hunter, Buckingham-Pi input budget, Gap Law, and Residual Law. Only
  an irreducible remainder with that receipt may extend the floor. Missing
  values or proofs produce structured refusals, never a development default.
- Every stage satisfies admit-the-alien. Earth, Mirror, the Sun, a fixed orbit,
  a familiar crust or atmosphere, and a solar hindcast are never implicit
  initial conditions. Sourced reference data may define a mechanism's validity
  domain, but cannot select a physical realization. The realization derives
  from the validated floor and named laws; seeded contingencies are generated
  inside the pipeline from derived measures.
- The declared abiotic arc is star/disk/system, assembly/composition,
  orbit/secular/moons, young thermal/materials, geodynamics/deep time,
  loads/flexure, then an immutable snapshot.
- Physical time and termination belong to the planet runner. Observer cadence,
  playback, camera, and rendering cannot advance or mutate planet state.
- A canonical viewer borrows only a completed immutable `PlanetSnapshot` and
  may transform that snapshot into presentation data.

The four tiers describe where a value resides, not four equivalent input
channels. Universal floor entries can enter preflight. Reference and Residue
entries enter the floor only through a derivation-exhaustion receipt. Written
state and Contingency are generated by the run; callers cannot supply them as
world values. A contingency draw must consume a derived measure and the seed.

The checked-in centralized inventory is
`docs/working/CANONICAL_LEDGER_INVENTORY.txt`. It is generated from the audited
catalog by `just ledger-inventory`; `just ledger-inventory-check` fails when the
file is stale. The inventory always contains the complete four-tier by
seven-provenance matrix, including zero-count cells, plus member identities.

## Parked evidence

The nested `parked/Cargo.toml` workspace contains the retired biology, compose,
and causal-viewer packages. `just audit-parked` runs their historical
constructor, provenance, profile, and quarantine checks. Those checks preserve
what the old arc claimed; they do not make it canonical and do not block the
planetary readiness result.

Legacy examples, scenarios, calibration data, and tests live in the nested
`parked/` workspace. Their values may be retained only as clearly named
fixtures, and no fixture value may be promoted into `civsim-planet` without a
canonical derivation from the absolute floor or an irreducible floor-admission
receipt.
