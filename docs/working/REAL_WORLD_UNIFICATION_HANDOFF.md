## 3. The crux: the being-identity bridge

Today `World` minds and `Embodiment` walkers are disjoint id spaces that would collide. `World` mints ids from its internal `Registry` via `self.reg.mint()` (world.rs:1156, 1199), monotonic from 0 in dawn seeding order. `Embodiment` walkers carry caller-assigned ids; the evolve harness hardcodes `StableId(1)` (evolve.rs:651), with no minting inside `Embodiment` and no shared registry. Because `body_temp: BTreeMap<StableId, Fixed>` and the `LocationIndex` key on the raw `StableId`, a `World` mind id N and an `Embodiment` walker id N alias the same body slot. A being that is both a located body and a cognition mind does not exist yet.

**Recommended decision (settle this first):** make `World`'s `Registry` the single minting authority for the union. Thread the `founders` vec that `seed_dawn_populations` returns into `Walker::new`, so one `StableId` owns an entry in both `World.minds` and `Embodiment.walkers`; never mint from a second registry. Add a unified constructor `Runner::with_world_and_embodiment` (world Some, embodiment Some), preserving the `!world.has_behaviour()` refusal (runner.rs:772). `step_inner` and `state_hash` already handle both Options, so the structural change is the constructor plus one scheduler edge.

The determinism consequences to hold:
- **state_hash** already folds `body_temp`, the world hash, and each id-sorted walker. A shared being now appears in all three, deterministic and id-ordered, but exercised nontrivially for the first time: verify each section folds once, no id collision, stable walker order.
- **RNG is safe** because draws are counter-keyed by `(locus, tick, Phase)` via `DrawKey::entity`. Locomotion draws on `Phase::EXPLORE` while cognition draws on LANGUAGE / MATE_CHOICE / CONVERSE, so the Phase discriminates the two streams of one being at one clock.
- **The two clocks differ by one within a tick:** `World.clock` increments at the start of `World::tick`, `Runner.clock` at the end of `step_inner` (the one locomotion keys on). Pin which clock keys a shared being and document it.
- **The sharp Principle 3 hazard is the scheduler.** `step_scheduled` may place `SYS_WORLD` in a parallel batch with the embodiment systems because `RES_WORLD` is declared disjoint from `RES_BODY`/`RES_INDEX` (runner.rs:947). That is bit-identity-safe only because world and embodiment share no being today. Once a being is both, both systems mutate its state but the resource declarations do not express the dependency, so the scheduler could reorder and diverge. Fix: add a shared resource id (a `RES_BEING` covering the union population) to both `SYS_EMBODIMENT` and `SYS_WORLD` so they serialize in the pinned order.

**On PlaceId versus Coord3:** cognition co-locates by an abstract `PlaceId = u32` (world.rs:83), embodiment by `Coord3` on the field. Full reconciliation (deriving PlaceId from the cell) is research-gated. Do not author it in this arc. Recommended: make `Coord3` the authoritative physical position, keep each being's `PlaceId` frozen at its dawn band, and accept that in this arc movement is thermoregulation-only while social geography stays at the dawn band. That is deterministic and honest, and it defers the seam cleanly.

**Lifecycle:** once reproduction and drift are armed and bodies are coupled, a birth (`World::reproduce`) must mint a paired body and a death (locomotion `alive = false` or `World` mortality) must retire the paired half before `state_hash` folds, or `body_temp`/index/minds desync.
