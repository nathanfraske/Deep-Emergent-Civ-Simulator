## The arc, and why the Verdict kernel leads it

The owner's materials oracle (`MATERIALS_ORACLE_SPEC.md`) is one flow, `K_solid(x_local; E; h, seed) -> {phases, modes, solution compositions, texture, properties, records, flags}`, across nine stages: the static data plane (0), the scenario plane (1), the proposer (2), class dispatch (3), the disposer (4), the freezer (5), property emission (6), aggregation (7), lazy ores (8), and the self-checks (9). It is the answer to the authoring-registry defect: the named-substance floor (granite density authored on the substance, the mineral registry, the formula column) is deleted, and materials become a pure function of composition, environment, path, and seed. That deletion is the same `derive_first_defect` debt Phase 2 made countable, discharged.

The owner's Verdict kernel contract (`VERDICT_KERNEL_CONTRACT.md`) is the spine of that flow, and it leads the buildout: the selection loop (proposer at Stage 2, disposer at Stage 4, freezer at Stage 5) is one protocol governed by the Gap Law, and reifying it as a type converts the meta-laws from convention-enforced to type-enforced before any physics rides on them. So slice 1 is the kernel CONTRACT, physics-free, and the thermochemical and attractor instantiations, then the property-emission stages, land on top of it. Building the contract first means every later stage inherits the discipline structurally rather than by reviewer memory.

Grounding the current substrate, verified at source: there is no Verdict, oracle, proposer, or disposer type anywhere in the codebase on any branch, so slice 1 is a clean new build, not a duplication. The one existing disposer-shaped code is `crates/physics/src/petrology.rs` (`stable_assemblage`, a Gibbs-minimization over the fixed phase registry, plus `apparent_gibbs_energy`); it is the limited Stage-4 seed the disposer instantiation will generalize, not the kernel. The determinism primitives the kernel needs already exist in `crates/core`: `canonical::canonical_sorted` (input-order-independent ordering) and `command::content_id` over a `StateHasher` (a deterministic content hash). The kernel reuses these rather than inventing its own, which is what makes the canonicalization law cheap to honor.

## The type reification (the Gap Law made unrepresentable-to-violate)

The core object is the verdict, and the typestate move is the whole point: a caller can read a winner only when the deciding model resolved one. Reified as a Rust enum generic over the candidate type `C`, so one contract serves both instantiations:

- **`Verdict::Decided { winner, runner_up, delta, resolution_s, band, provenance_key, tie_slot }`**: the deciding model separated the winner from the runner-up by `delta >= resolution_s`. The winner is readable here and only here (with `Trivial`).
- **`Verdict::Escalate { candidates, delta, resolution_s, provenance_key }`**: `delta < resolution_s`. There is NO `winner` field to read. The near-degenerate set escalates up the provenance ladder (a better-pinned energy model, per rule 2 of the Phase-2 forward-guard contract).
- **`Verdict::SeededDraw { drawn, tie_slot, provenance_key }`**: `delta` collapsed toward zero (a sub-resolution tie the ladder cannot break). The pick is a seeded contingency draw keyed on the candidate's content hash, occupying the named tie slot, and it is a counted authored draw in the honesty accounting.
- **`Verdict::Trivial { winner }`**: a single-candidate query where the physics is unambiguous, so the ceremony is cheap, but it is LOGGED so ceremony-avoidance is itself auditable.

The construction is sealed: `Decided` and `Trivial` are the only variants exposing `winner`, and only the `dispose` function (below) constructs a `Verdict`, so a caller cannot fabricate a `Decided` with `delta < resolution_s`. The resolution-ladder rule stops being a rule anyone remembers and becomes a state that cannot be constructed. This IS rule 2 of the Phase-2 consumer/form-side spec reified, and per the same-slice obligation there, the guard (the typestate) lands with its first client (the disposer instantiation), so slice 1 defines the type and the disposer slice wires the first real dispose.

## The correct factoring: two pure functions and a fold

Per the owner's contract, the loop is two pure functions and a fold, not a three-stage monolith:

- **`Proposer::propose(x, E, seed) -> Vec<C>`**: a pure function of composition, environment, and seed, memoized on quantized keys (composition class, environment bucket). It enumerates viable candidates.
- **`Disposer::dispose(candidates, E, seed) -> Verdict<C>`**: a pure function that assembles the energy per candidate and returns the verdict whose variant structurally encodes whether a winner is readable. It owns the `delta`-versus-`resolution_s` decision.
- **`quench(equilibrium_fn, path) -> RealizedState`**: the freezer is NOT stage three. It consumes the path `h`, so it is a functional over trajectories, owned by the time-marching layer as a fold, never inlined into the pure oracle (inlining would smuggle history into a state function and break the solver law's memory clause). Equilibrium is a state function; realization is a path functional; the type signatures keep them separate.

`preflight(x, E) -> Validity` runs the representation theorems (the reference-validity checks the owner names: the boolean, the class window, the U-over-W window, the Bohr-van-Leeuwen-class checks) BEFORE `propose` is ever called, so a reference-validity failure is a compile-path `Validity::Invalid(reason)` rather than a review-path mistake. In slice 1 the theorem bodies are the instantiations' to fill; the kernel defines `Validity` and the `preflight` seam.

## The two engineering laws, reified onto the core primitives

Both are descendants of banked landmines, and both wire to primitives that already exist:

- **Candidate canonicalization.** A seeded draw must key on a content hash of the candidate, never on enumeration order, or a proposer refactor silently reshuffles every tie-broken world in the shipped universe (the byte-neutrality landmine one level down). The kernel requires `C` to yield a deterministic content hash (via `core::command::content_id` over the candidate's canonical fields), sorts candidate sets with `core::canonical::canonical_sorted` before dispose, and indexes the seeded draw by content hash. The hot disposer paths monomorphize (they sit in inner loops under Q32.32 determinism).
- **Coverage is audited, not proven.** `delta` is only as honest as the proposer's coverage: a gap computed against an incomplete candidate set is false confidence (the CO lesson, where valence arithmetic never nominated the winner). The type system cannot fix this, so the kernel carries the audit hooks: the Verdict log is a queryable record (counts-are-queries, so "how many authored draws did this world consume" is a filter over the log, and the near-degenerate map is a query over `Escalate`/`SeededDraw` verdicts), and the coverage spot-check interface (a seeded structure search that the winner is not beaten by anything outside the proposed set) plugs in at the instantiation. The Residual Law's disequilibrium clause and periodic compute-once sampling are the other two mitigations, wired at their stages.

## The free payoffs, and the coupling to Phase 2

Every Verdict carries a `provenance_key`, and that key references the joined provenance register Phase 2 landed (`civsim_sim::unified_provenance`), so provenance-keyed covariance and the honesty accounting ride the same key: a seeded authored draw is a counted entry on the authoring surface, queryable rather than remembered. This is the coupling that makes the honesty number live through the materials buildout: as a stored-measured floor value (a `derive_first_defect`) is replaced by a derived Verdict output, the defect clears and the number improves, and the Verdict log is where that shows up.

## The seams for the gate (rule before I build)

1. **Where the kernel lives.** The contract is generic (no physics) and determinism-flavored (it uses the `core` canonical and content-hash primitives), while the two instantiations are physics (near `petrology.rs`). Three options: a new module in `crates/core` (alongside `canonical.rs`, since the kernel is a determinism primitive), a new module in `crates/physics` (near the disposer it will host), or a dedicated `crates/materials` crate for the whole substrate (kernel plus instantiations plus stages), keeping `physics` for the floor. My recommendation is the dedicated `crates/materials` crate: the substrate is large (nine stages), it depends on `core` and `physics` but is a distinct layer, and a crate boundary keeps the plugin-trait discipline honest (the kernel crate cannot see the floor's internals). If you prefer the lowest-friction start, a `crates/physics/src/verdict.rs` module lands slice 1 with no new crate wiring, and the crate extraction becomes a later slice. Your call on which.
2. **The candidate genericity.** `Verdict<C>` generic over the candidate type serves the two instantiations (a phase assemblage for thermochemical, an attractor state for the attractor kernel) with one contract. The alternative, a concrete candidate enum, would author a closed list of candidate kinds, which is the templating Principle 8 forbids. I read the owner's "one contract, two instantiations" as mandating the generic. Confirm.
3. **The `provenance_key` type and its register coupling.** The key references the joined register (`unified_provenance`), which lives in `crates/sim`. If the kernel lives in `core` or `physics` (below `sim`), the key cannot be a `sim` type without an inversion. Options: the key is an opaque interned id (a `u64` content id or a small newtype) that `sim` resolves against the register, keeping the kernel below `sim`; or the honesty coupling is a `sim`-side query over the Verdict log rather than a field the kernel resolves. I lean toward the opaque interned id (the kernel carries the key, `sim` owns the register lookup), which keeps the dependency direction clean. Confirm the direction.
4. **`delta`, `resolution_s`, and `band` as `Fixed`.** All three are Q32.32 `Fixed` (the deciding model's energy resolution and the gap are energies in the same units). The `band` is the error band on the winning energy (a `Fixed` half-width, or a low/high pair). Confirm the band shape (single half-width versus an explicit interval).
5. **The memo's determinism and placement.** The proposer and disposer memoize on quantized keys, and the memo must iterate in canonical order (never insertion order) to stay byte-neutral, sitting outside the pure functions (a memo is a cache, not state hashed into the world, R-CANON-WALK). Confirm the memo is a kernel-provided, canonically-iterated cache rather than each instantiation rolling its own.
6. **The `Trivial` log and the coverage hooks in slice 1.** Slice 1 defines the log and the query interface but has no verdicts to log yet (no instantiation). Confirm slice 1 lands the log structure and its queries with synthetic-verdict tests (proving the queries work), and the first real verdicts arrive with the disposer instantiation, same-slice with rule 2's guard.
