# Physics substrate audit registry (what exists, and what must derive)

This is a derive-first audit and discovery map, not an admission list. The authoritative canonical membership is `docs/working/CANONICAL_LEDGER_INVENTORY.txt`, generated from the repository-owned absolute-floor catalog. A citation, provenance tag, stored axis, substance row, or appearance in this report never authorizes a value at the planet front door. This report inventories candidate reference columns and every floor LAW with its file:line so it doubles as the "where do I look for this law" index. The DERIVING-SUBSTRATE half (the section right below the constants) is every subsystem OUTSIDE the physics crate that derives a world quantity from the floor and the situation: orbital mechanics, the hydrology cycle, productivity, metabolism, the matter cycle, the time-space anchors. The derive-vs-author rule it makes concrete: a value must first DERIVE from the absolute floor and situation, and the deriving-substrate list is where you look for the substrate that derives it BEFORE you ever author a number. If a quantity you were about to author appears in the deriving-substrate list, authoring it is a defect. Only an irreducible remainder that survives the derivation hunter, Buckingham-Pi budget, Gap Law including its typed Chaos Protocol branch, and Residual Law may extend the floor. Written state and contingency are generated inside the run from admitted physical measures and the approved realization-coordinate law, never supplied as world values or caller seeds.

There are 89 stored material and quantity AXES across 36 candidate-reference files. The same files carry 14 reference SUBSTANCES as vectors over those axes. These are migration and derivation-hunt populations, not automatically admitted floor values. The floor laws are 70 DECLARED laws (the `[[law]]` blocks in the floor data) plus 109 direct kernels in `crates/physics/src/laws.rs` (39 of them not backed by a declared block, called called directly), all listed below. A law kernel is fixed Rust, not a value-bearing input; every magnitude it consumes still owes the absolute-floor admission contract.

The lists below are GENERATED from `crates/physics/data/*.toml`, `crates/physics/src/laws.rs`, and every `// @derives:` marker across `crates/**/*.rs` by `scripts/gen_floor_registry.py`, with each entry's `file:line`, so they never drift. Do NOT edit this file by hand: change the floor data, add the law kernel, or place the `// @derives:` marker at the deriving subsystem, then run the generator (the floor-registry stop-gate regenerates and blocks if this file is stale). To ADD a floor axis, a law kernel, or a deriving substrate is a deliberate act; the diff to this registry is the record of it. When you build a new subsystem that derives a world quantity, mark it with `// @derives: <quantity> <- <from these inputs>` so it lands on this billboard and no future agent authors from the ether what your subsystem already derives.
