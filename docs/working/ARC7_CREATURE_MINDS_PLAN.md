# Arc 7 (creatures-have-simpler-minds) first-slice build plan

From the ultracode scoping workflow (2026-07-08, map + high-effort design, verified against source). KEY FINDING: the controller perceives ONLY its own reserves + static field percepts, NOT other beings, so flee/chase cannot emerge until a being-perception percept is added to the shared ControllerLayout (a re-pinning DEFERRED slice). The first slice makes creatures LIVE (spawn as walkers, move, forage, metabolize, die) reusing the founder machinery, byte-neutral behind a new opt-in flag.

## first_slice

Spawn exactly the occupant set `LivingWorld::consumer_bodies` (genesis.rs:280-318) already iterates today (the one-representative-per-surviving-species-per-region non-autotroph placements from `place_survivors`, genesis.rs:886-937) as real `Walker`s instead of static deposited tissue, riding the SAME `Embodiment`/`step_embodiment`/locomotion loop every founder already rides, with NO new percept, NO new loop, and NO species/race branch in the hot path. Concretely: for each such occupant, mint an individual genome off its species' `GenePool` (`pool.promote`, already proven at biosphere.rs:505), grow its body via the SAME two-tier branch `assemble_dawn_embodiment` already runs for founders (catalog `body_plan` vs Arc-6 grown `Structure`, worldbuild.rs:565-590, unmodified), express its controller via `Controller::express` against the run's one shared `ControllerLayout` (unmodified), and call the SAME `Walker::new`/`Embodiment::add` a founder uses (unmodified). This alone makes every one of those creatures perceive its own reserves and the matter field, move, forage, metabolize, and die under the existing embodiment mechanics, no different in kind from a founder walker minus the mind/reproduction apparatus. The ONE piece of genuinely new mechanism required, not deferrable, is a Controller gene block on the species genome (species pools today carry zero `Channel::Controller` loci, a confirmed gap: `morphogen_gene_set_with_prefix`, morphogen.rs:505-535, only ever wires niche and morphogen loci), because without it every promoted creature genome expresses an inert, permanently-zero controller. Deferred to named later slices because they need mechanism this slice cannot cleanly carry: flee/chase (needs a new physics-floor percept for sensing another being, which does not exist anywhere in the substrate today, confirmed by direct read of controller.rs/locomotion.rs, and is an owner-facing value-authoring-line question, not a code-plumbing one); creature reproduction/selection (needs a leaner sibling of `World::reproduce`'s heavy band/mate/sex machinery, since today a species has exactly one static individual per region that is never reborn); and Great Beasts (a data dial, cheap once slice 1's gene-block pattern exists, but naturally rides on top of it rather than gating it).

## emergence_not_authored

Confirmed by direct read (locomotion.rs 839-943, controller.rs 15-56/83/141-175): the ONLY percepts a Walker's controller reads today are its own reserve levels, a same-tile source flag, a direction-to-nearest-known-point-source, a signed setpoint deviation, and optional opt-in feature/appetitive/material/attraction blocks, all keyed on tile/matter composition (ResourceField::source, locomotion.rs:354-368) or belief state. There is NO being-presence, species-identity, or relative-motion percept anywhere in the substrate; the only inter-being read is Walker::observed_actions (locomotion.rs:624-632), a valence-free action label for imitation learning, never a threat signal. This means flee/chase genuinely cannot emerge from the first slice as designed: a creature spawned by this slice perceives its OWN homeostatic state and the matter field exactly as a founder does, and if two creature species are mutually predator/prey, the ONLY channel through which that relationship could show up today is the existing edibility/geophage mechanism (one being's tissue is eaten as located matter when another stands on it and forages), which is passive matter-consumption, not agentic pursuit or evasion. Getting flee/chase to emerge honestly requires a NEW physics-floor percept: a general "nearby-mass-in-motion" or "nearby-hostile-matter" axis that a body's own physiology/tolerance defines (so it keys on the being's own data, per Principle 9's alien-feasibility and derive-vs-author lines, never on a hardcoded "predator" tag), riding the SAME known-source/signed-deviation percept plumbing every other axis already uses (map notes point 6, "not gated on new engine mechanism per se, but on what physics-floor primitive threat/predation keys off"). That is a value-authoring-line question for the owner (what floor axis backs it, what its extraction/threat-derivation basis is), not something this slice can build. The first slice therefore delivers only the "creature is a living agent" half (spawn, perceive-its-own-state, move, metabolize, forage, die) and explicitly defers flee/chase to a later slice gated on the owner resolving that new percept primitive; selection over MANY generations of creature turnover (needed for any flee/chase controller weight to be found and fixed by fitness) is itself deferred to the reproduction slice (creatures today are one static individual per species per region, never reborn), so even once the percept exists, flee/chase cannot emerge until creature reproduction (slice 3) exists to select over it.

## reuse_vs_new

Reused UNMODIFIED: Homeostasis::new/::from_structure, grow()/Structure::digest(), Controller::express/::zeros, Physiology::express, Walker::new/with_structure, Embodiment::add (a bare Vec push, runner.rs:939/2685), the whole step_embodiment/locomotion/perceive loop (locomotion.rs 839-943, 968+), GenePool::promote (genome.rs:1079, already proven for representative_structure), and the one-occupant-per-surviving-species-per-region placement genesis.rs already computes (place_survivors, 886-937) plus the exact (coord, composition, volume) tuple consumer_bodies already derives (280-318) as the natural spawn seed. Genuinely NEW: (1) a Controller gene block on the species genome (species pools today carry only niche_loci + morphogen loci, morphogen.rs:505-535, with no Channel::Controller loci at all, confirmed gap, so a promoted individual currently expresses an inert unseeded controller); (2) the spawn_creature_walkers assembly function itself (a lean sibling of assemble_dawn_embodiment, worldbuild.rs:493-645, stripped of the band/PlaceId/lifecycle-kit spawn-template machinery a founder needs since a creature is not reborn by this slice); (3) a disjoint StableId namespace for creature walkers (no existing minting path for "many individuals under one species," only the single fixed `(region_id<<20)^species_id` representative id, genesis.rs:908); (4) the great_beast data dial on Species. A creature is cheaper than a founder for two reasons that fall out of the existing architecture for FREE, not from new leanness code: exploration/deliberation/social_learning are already founder-zero-gated (`if w.exploration > Fixed::ZERO` etc., runner.rs:5077/5034/4228) so simply never wiring those channels into a species' gene set costs nothing extra per tick; and a creature carries no World-mind, mate-pairing, or band apparatus (World::reproduce, world.rs:2346, is skipped entirely), so it never enters the heavy founder reproduction path. Per-tick perception cost (perceive()'s sense_range-square scan, locomotion.rs:839, default 4) is IDENTICAL per walker regardless of species since sense_range is a single shared LocomotionParams value, not race/species-keyed, so the real cost delta this slice buys is fewer channels expressed and no reproduction pairing, not a cheaper perception loop.

## byte_neutral_path

Gate the whole slice behind one new opt-in flag (default off, parallel to the existing `cfg.full`/`corpse_matter` pattern at run_world.rs:1868-1886) that swaps the current `consumer_bodies -> emb.tissue_mut().deposit` static-matter loop for `spawn_creature_walkers`. Flag off: the run takes the byte-identical current path, nothing new is constructed, no new RNG phase is drawn, hash unchanged (verified by the same "opt-in, hash-neutral by default" pattern already documented throughout locomotion.rs/runner.rs for exploration, delta percepts, carried matter, etc., e.g. locomotion.rs:536-538, 590-592). Flag on: the run diverges (new StableIds minted, new controller-gene RNG draws via DrawKey::entity keyed on the new creature ids, a changed state_hash), which is expected and fine since it is a strict opt-in, same as `--scenario full` itself. What re-pins if this slice is later extended: the moment step 7's great_beast dial or the flee/chase percept (deferred) touches the SHARED ControllerLayout width (a new percept block added to `ControllerLayout`, like the feature-percept block already does at worldbuild.rs:527-533), EVERY walker in the run, founder or creature, re-expresses against a wider layout and the whole run's hash changes even with the new mechanism unused, exactly as the existing code comment at worldbuild.rs:527-533 already warns for the toxin-percept block. So this slice's own gene-set and spawn additions are safe (creature-scoped, no shared-layout width change), but the deferred flee/chase percept slice is NOT byte-neutral against founders by default and must ship its own explicit re-pin plan (e.g. only widening the layout when the world declares a nonempty threat-percept registry, mirroring PerceptRegistry::from_tolerances' "a world with no declared toxins declares no percepts, so the layout is unchanged" guarantee, worldbuild.rs:526-533).

## great_beasts

The richer-mind exception is a single `great_beast` (or tiered) field on `Species` (biosphere.rs:161-176), read only once, at spawn time in step 3/7 of the build steps, to decide whether that species' walkers get the Exploration/Deliberation/SocialLearning gene channels wired (step 1's controller-gene-set builder called with vs without those extra blocks) and a wider `sense_range`/deeper percept blocks. This reuses the EXACT mechanism founders already use for their own opt-in richness (worldbuild.rs:615-625 wires exploration/deliberation/social_learning per founder from its own genome; the founder-zero pattern already means "richer" and "leaner" differ only in which channels a gene set carries, never in a branch of the step_embodiment loop itself, since that loop already treats every walker's ideation fields as ordinary opt-in Fixed values regardless of species). So Great Beasts cost more only because their gene set is bigger and their walker carries live (non-founder-zero) exploration/deliberation weights that route through the ALREADY-EXISTING discovery/deliberation passes (runner.rs 4764-5095) other walkers with zero weight skip via the early-out checks; no new code path, no RaceId/SpeciesId branch in the hot loop, purely a data flag deciding which gene blocks get built for that species (Principle 9 compliant: the mechanism is uniform, the richness is data).

## risks

Determinism: creature StableIds must be minted in a namespace PROVABLY disjoint from founder mind ids (build step 4) since `Embodiment::walkers` is a bare Vec (runner.rs:939, no collision guard) but `reconcile_lifecycle` and every `DrawKey::entity(id, ...)` RNG draw key on raw StableId, so an accidental collision would silently cross-wire a founder's and a creature's RNG streams or death handling, a correctness bug that would NOT show up as a panic, only as a subtly wrong replay; must be caught by an explicit fail-loud disjointness assertion, not assumed safe. Performance: this slice spawns one Walker per surviving species per region (the existing place_survivors granularity, genesis.rs:886-937), which for a biosphere with hundreds of species across many regions is potentially hundreds to low thousands of extra walkers, each paying the full perceive() sense_range-square scan (locomotion.rs:839, same cost as a founder) every tick; this is bounded and known (not a population explosion, since there is still exactly one individual per species per region, no reproduction yet), but should be measured against the existing perf-profiling baseline (perf-profiling-findings memory: regrow allocation, Rayon threshold) before landing, since it multiplies the per-tick embodiment cost by roughly (founders + surviving-species-count) rather than founders alone. Biggest correctness/authoring risk: the Controller gene block (build step 1) must be threaded through the SAME `ControllerLayout` every founder expresses against (build step 2), or a creature's expressed controller silently reads misaligned weight indices (the exact bug morphogen_gene_set_with_prefix's prefix trick was built to prevent for the morphogen block, morphogen.rs:505-518 doc comment); this is the same class of index-alignment defect the codebase has already been bitten by once (per that doc comment) and is the single highest-risk step in the build order. Second-biggest risk: without live creature reproduction (deferred to a later slice), a species' single spawned individual, once dead, is simply gone from the world for the rest of the run (no replacement), so the food web's turned "lives" are a one-shot event per species per region rather than an ongoing population process; this is an honest, named limit of the first slice, not a hidden defect, but must be stated plainly wherever the slice is presented so it is not mistaken for full population dynamics.

## Build steps (first slice)

1. 1. Wire a Controller gene block onto the species genome: add `controller_gene_set_with_prefix(prefix_loci, layout)` in evolve.rs (or morphogen.rs, sibling to `morphogen_gene_set_with_prefix` at morphogen.rs:505-535), mirroring `controller_gene_set` (evolve.rs:238-250) but offset past the species' existing loci. In biosphere.rs, wherever a species' `GenePool`/gene-set is constructed (the niche_loci + morphogen block), append this controller block so the full species gene set is [niche_loci][morphogen block][controller block], index-aligned like the existing niche/morphogen split. Without this the express reads unseeded loci forever (inert, confirmed gap).

2. 2. Extend `RegionBiosphere`'s per-species state (or a value threaded alongside it) with the `ControllerLayout` the embodiment's `Embodiment::layout()` produces (worldbuild.rs:534), so `genesis()` can build each species' controller gene block against the SAME shared layout every founder Walker expresses against (one input/output shape for the whole run, no per-species divergence, Principle 9). Concretely: thread `&ControllerLayout` into `genesis()`/`GenesisParams` (currently `genesis()` at genesis.rs:514 takes seed/params/abiotic/morphogen only) so species gene sets are built once the layout is known, or defer gene-set construction to a post-genesis pass in worldbuild that mirrors `assemble_dawn_embodiment`.

3. 3. Add a `spawn_creature_walkers` function in worldbuild.rs, a lean sibling of `assemble_dawn_embodiment` (worldbuild.rs:493-645), scoped to exactly the occupant set `LivingWorld::consumer_bodies` already iterates (genesis.rs:280-318: non-autotroph occupants, i.e. `place_survivors`'s one-per-surviving-species-per-region placements, genesis.rs:886-937). For each occupant: (a) mint the individual genome via `sp.pool.promote(seed, individual_id, ploidy)` (genome.rs:1079, already proven at biosphere.rs:505 for representative_structure); (b) grow body + reserves by the SAME two-tier branch `assemble_dawn_embodiment` already runs (worldbuild.rs:565-590): `Homeostasis::new` for `body_plan: Some`, `representative_structure`-style `grow()` + `Homeostasis::from_structure` for `body_plan: None`; (c) express the controller via `Controller::express(&species_gene_set, &genome, &layout)` (controller.rs:780-838, unmodified); (d) build `Physiology::express` the same way founders do (worldbuild.rs:598-608), reading the SAME `HomeostaticRegistry`/`ToleranceRegistry` every walker already reads; (e) leave `walker.exploration`/`deliberation`/`social_learning` at their Walker-default (zero) rather than expressing them, since a species gene set with NO Exploration/Deliberation/SocialLearning channel loci reads inert (matches the existing founder-zero pattern, no new mechanism needed for leanness) unless the species is flagged as a Great Beast (step 6).

4. 4. Mint each creature's `StableId` in a namespace PROVABLY DISJOINT from founder mind ids: fold a fixed high discriminator bit (mirroring `OccupantId::organism`'s `OccupantKind::Organism` tag, located.rs:41-70) into the id passed to `Walker::new`, e.g. `StableId(occ.id.id.0 | CREATURE_ID_TAG)`, and add a debug-fail-loud assertion at spawn that no creature id collides with any founder id (`world.being_ids()`), since `Embodiment::walkers` is a bare `Vec<Walker>` (runner.rs:939) with no key-collision protection at the container level, but `reconcile_lifecycle` and every RNG `DrawKey::entity(id, ...)` call DO key on raw `StableId` and would silently cross-wire a founder and a creature that share one.

5. 5. In `Runner::step_embodiment` / `reconcile_lifecycle` (runner.rs:4536, 5214), confirm (by reading, then a targeted test) that a creature Walker with no backing `World` mind is handled correctly on death: `reconcile_lifecycle`'s newborn-embodiment and death-propagation logic (5236-5289) is written against a `World` mind driving `world.remove_being`; a creature has no such mind, so its death path must be the SIMPLER branch already used for any walker with no cognition-side counterpart (confirm this branch exists and is exercised for a walker whose id has no `World::being` entry, or add the minimal guard that skips the world-side reconcile step for a creature and just drops it from `emb.walkers` plus, if `corpse_matter` is armed, deposits its tissue exactly as any other retiring walker already does at 5257-5283).

6. 6. Wire the `--scenario full` run path (run_world.rs:1864-1893): behind a NEW opt-in flag (default off), replace the `consumer_bodies` -> `emb.tissue_mut().deposit` static-matter loop with a call to `spawn_creature_walkers`, passing the same `living.consumer_bodies`-equivalent occupant set so no body's matter is double-counted (a creature's tissue is now carried on its own grown Structure/BodyPlan, not pre-deposited as static tissue). When the flag is off, the exact current static-deposit path runs untouched (byte-neutral).

7. 7. Add a `great_beast: bool` (or richer tier enum, data-defined per Section 11's registry-not-enum discipline if more than two tiers are ever needed) field on `Species` (biosphere.rs:161-176), read ONLY at spawn to decide whether step 3(e)'s ideation channels and a wider `sense_range` are wired for that species' walkers, exactly mirroring the founder opt-in pattern (worldbuild.rs:615-625) with zero new plumbing, so the lean/rich split is a data dial, not a second code path.

8. 8. Run the full verification suite (design-doc + audit-log em-dash/adverb/parts/fence checks do not apply here since this is source code, not the design document; instead: `cargo test -p civsim-sim` for the new gene-set/spawn functions, a determinism replay test mirroring `genesis_replays_bit_identically` (genesis.rs:942-957) extended to creature-walker state hashes, and a byte-neutrality test asserting the flag-off run hash is unchanged versus the pre-slice baseline).


## Map

MAPPING: founder-walker pipeline vs. biosphere-creature-as-static-matter gap, grounded against source.

## 1. How a founder becomes a living Walker (the target shape)

`crates/sim/src/worldbuild.rs::assemble_dawn_embodiment` (fn at line 493, called from line 405) builds one `Embodiment` shared by the whole world and, per founder StableId, assembles:
- **Body + homeostasis** (lines 578-590): if `race.body` is `Some` (catalog tier), `Homeostasis::new(&genesis.homeostatic, plan, &genesis.organs)`; if the race carries `race.morphogen` instead (Arc-6 grown tier), a `Structure` is grown from the founder's own genome (`grow(program, &params, seed, id)`, lines 565-571) and `Homeostasis::from_structure(&genesis.homeostatic, s)` sums reserves off the grown tissue directly. A race with neither is skipped (line 553-555).
- **Controller** (line 591-594): `Controller::express(&race.genes, genome, &layout)` when the founder has a genome, else `Controller::zeros(&layout)`. `layout` (line 535) is ONE `ControllerLayout` shared by the whole `Embodiment`, built once from the world's `HomeostaticRegistry` + `AffordanceRegistry` + `PerceptRegistry` (line 534) — every walker in the run, whatever its race, is expressed against this same input/output shape.
- **Physiology** (598-608): `Physiology::express(&genesis.homeostatic, &genesis.tolerances, &race.genes, genome)`, the per-class nutrient requirement/assimilation a forage bite is measured against.
- **Placement**: a deterministic `Coord3` per band (line 556), recorded in `spawn_by_place` so a newborn of that band's lineage spawns at the same site (lifecycle-kit template).
- **Ideation channels** (615-625): `exploration`, `deliberation`, `social_learning` expressed from genome via unit evolve-channels, founder-zero by default.
- `Walker::new(id, coord, body, homeostasis, physiology, controller)` (locomotion.rs:640, struct at 503) then `.with_structure(s)` if grown (locomotion.rs:682), then `emb.add(walker, thermal)`.

Per tick, `Runner::step_embodiment` (runner.rs:4536) does: (0) reads world-physics-derived percepts (temperature gradient, avoidance/attraction gradients, appetitive belief) per being into `BTreeMap<StableId, ...>`s, all pure reads keyed by `w.id`, no RNG; then drives `locomotion::step_with_field_dirs` (locomotion.rs:968) which per-walker: `perceive()` (locomotion.rs:839, walks a `sense_range` — default 4, LocomotionParams:113/183 — square of tiles around the being and calls `resources.source(axis, c, homeo)` to learn point sources into `w.known`), builds the controller input, `Controller::evaluate`, decides an affordance (Move or Ingest/geophage), applies metabolism drain, and kills the being if a reserve crosses its floor (`w.alive = false`, checked ~line 1286-1290). `Runner::reconcile_lifecycle` (runner.rs:5214) then reconciles bodies against the cognition `World`: a locomotion death propagates to `world.remove_being` (5236-5238); a retired body optionally deposits corpse tissue (`emb.tissue.deposit`, 5257-5283, gated by `self.corpse_matter`); a newborn `World` mind whose race carries a body plan is embodied at its band's frozen spawn site (comment at 5287-5289, mirrors the dawn assembly). **Births and deaths are driven by the cognition `World`, not by locomotion**: `World::reproduce` (world.rs:2346) groups mature beings by `PlaceId`/band (2366-2382), pairs them by mate compatibility and sex within a race (2384-2429, `choose_mate`, `sexes_compatible`), then bears one child per pair (2434+). This is heavy, race/band/sex-typed machinery.

## 2. What the controller actually perceives today (the central finding)

The controller's inputs (controller.rs, doc comment 29-41, `INPUTS_PER_AXIS = 5` at line 83) are, per homeostatic axis in canonical order: reserve level, a "source of this axis on my tile" flag, the 2-component unit direction to the nearest **known point source**, and a signed setpoint-deviation percept — plus optional feature/appetitive/material/attraction blocks (all opt-in, all reading substance composition or belief state, `ControllerLayout` fields at 141-175) — plus a bias. `perceive()` (locomotion.rs:839-852) and `ResourceField::source` (locomotion.rs:354-368) both key exclusively on **tile composition** (`self.composition(coord).nutrient(class) > 0`), i.e. matter concentration in a `ResourceField` cell. There is no notion of another Walker as a sensed object anywhere in this pipeline: no being-presence percept, no species-identity percept, no relative-velocity/distance-to-another-agent input. The single existing crumb of inter-being sensing is `Walker::observed_actions` (locomotion.rs:624-632), a transient, valence-free set of primitive ids that co-located OTHER beings ate last tick, read only by the social-learning discovery sampler — it never carries who/what the other being is, only what action it performed, and even this exists purely for imitation-learning among founders, not predator/prey detection. Grep confirms: "predator"/"prey"/"flee"/"chase" occur only in body.rs wound-mechanics tests (a static strike-contest primitive keyed on natural weapons, body.rs:1503-1568), morphogen.rs affordance-registry fixture naming, biosphere.rs trophic-label derivation (a pure food-web READ, not a live perception), and a controller.rs doc-comment naming the eventual goal (line 183) — none of it is a live agent-sensing percept a Walker's controller reads. **Confirmed: today's controller perceives only its own reserves plus static field/tile percepts (matter, belief, gradients); it does not perceive other beings at all**, alive or dead, predator or prey.

## 3. What a founder Walker needs that a biosphere species has or lacks

Biosphere creatures currently live entirely in `crates/sim/src/genesis.rs`/`biosphere.rs` as a **population abstraction**, never as individuals:
- `LivingWorld.occupants: LocationIndex` (genesis.rs:116) places `OccupantId`s at `Coord3`s; `occupant_info: BTreeMap<OccupantId, OccupantInfo>` (117) carries only `species: SpeciesId, layer, region, body_mass: Fixed` (OccupantInfo struct, genesis.rs:101-107) — **no StableId, no individual genome, no position finer than the placed coord, no reserves, no controller**.
- `Species` (biosphere.rs:161-176) carries `body_plan: Option<BodyPlan>` (Some = catalog tier, None = Arc-6 grown tier), `draws_on`, and `pool: GenePool` — but `GenePool` (genome.rs:805-825) is an **allele-frequency population tier** (`freqs: Vec<Fixed>`, `effects: Vec<Fixed>` per locus), not a set of per-individual `Genome`s. It can mint one via `GenePool::promote(seed, individual_id, ploidy)` (genome.rs:1079), which the Arc-6 `representative_structure` (biosphere.rs:496-509) already uses — but only ONCE per species, at a fixed `species_stable_id(species_id)`, to grow a single display body. There is no standing per-individual genome store, no mutation-and-inheritance loop, no birth/death of individuals in this substrate today.
- **Critical gap in the gene set itself**: `morphogen_gene_set_with_prefix` (morphogen.rs:519, the function biosphere.rs uses to build a species' expressible `GeneSet`) only wires `niche_loci` prefix genes plus `Channel::Morphogen(...)` params (morphogen.rs:528-535). It **never adds `Channel::Controller(ControllerParamId)` loci** the way `evolve.rs::controller_gene_set` (234-250) does for founders. So even if an individual genome were promoted from a species' `GenePool` today, expressing it against the shared `ControllerLayout` via `Controller::express` would read only unseeded (zero) loci for every controller weight — behaviorally inert, worse than founder-zero because it can never be lifted by selection (the loci don't exist to mutate).
- `--scenario full`'s canonical run path (run_world.rs:1690-1701) currently calls `genesis(seed, params, abiotic, None)` — the fourth argument is the `Option<&MorphogenProgram>` that arms the Arc-6 grown tier (genesis.rs:514-541); passing `None` means **the canonical biosphere is generated catalog-tier only**, every species using its static sampled `BodyPlan`, not a grown `Structure`. Arc 6 landed the mechanism but the run path never arms it.
- What IS reusable as-is: `Homeostasis::new`/`Homeostasis::from_structure` (both call sites already proven for founders), the `grow()`/`Structure::digest()` machinery, `being_afforded`/`afforded_structure` (homeostasis.rs:852) deriving an anatomy's available actions from its own grown parts (never a species/race id), and the shared `Embodiment`/`ControllerLayout`/`ResourceField`/`TissueField` substrate every walker already rides regardless of race. `consumer_bodies` (genesis.rs:280-318) already computes exactly the per-individual, per-coord `(coord, composition, volume)` that would seed a walker's body and location — today it only feeds `emb.tissue_mut().deposit` (run_world.rs:1877-1886), static matter for the geophage, never a `Walker::new`.

## 4. The smallest set of new pieces to spawn a creature as a living walker

1. **A Controller gene block on the species genome**: extend the species gene-set builder (a sibling of `morphogen_gene_set_with_prefix`) to also carry `Channel::Controller(ControllerParamId)` loci per weight the shared `ControllerLayout` needs, the same pattern `evolve::controller_gene_set` already establishes for founders, so a promoted individual genome expresses a real (initially near-founder-zero, mutation-liftable) controller rather than reading unseeded zero loci forever.
2. **Per-individual genome minting at spawn, keyed off the species `GenePool`**: a leaner sibling of `World`'s birth path — `pool.promote(seed, individual_id, ploidy)` per spawned creature (mechanism exists, `demote`/mutation-on-birth for the reproduction loop needs wiring) — without the founder apparatus of bands, mate-preference, sexes, or `PlaceId`, matching the owner's "simpler/leaner minds" directive. This is the one genuinely new subsystem; nothing today mints or ages many individual genomes off one `GenePool` over generations.
3. **Individual placement + StableId minting for biosphere occupants**, replacing the current `OccupantInfo` (species/layer/region/body_mass only, no identity) with something that can back a `Walker::new(id, coord, body, homeostasis, physiology, controller)` call, reusing the already-proven `Homeostasis::new`/`::from_structure`, `grow()`, and `being_afforded` paths.
4. **Arm the Arc-6 grown tier on the canonical run path** (pass a real `MorphogenProgram` instead of `None` at run_world.rs:1690-1701) if grown, genome-driven bodies are wanted for creatures rather than static catalog `BodyPlan`s — orthogonal to (1)-(3) but needed for genome-to-body coherence.
5. **A lean death/removal + optional corpse-deposit hook** for these new walkers — `reconcile_lifecycle`'s corpse-matter deposit (runner.rs:5257-5283) and `emb.tissue.deposit` are already generic over any `Walker`, so this is wiring, not new mechanism, once (3) exists.
