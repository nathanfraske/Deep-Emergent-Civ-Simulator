# Affordance-percept hardening: the blind framing report (Agent B, sibling piece off Arc 3)

This is the doc-only opener for the `AffordancePerceptKind` Principle-8/11 hardening, the first flagged independent sibling substrate assigned to Agent B off the Arc-3 perception work (gate direction on PR #109). It carries the blind framing result for the gate to rule on before any code. No code is built here.

## The assignment as confirmed

The gate confirmed the scope on #109 as: (1) BUILD a data-source naming path for the closed `AffordancePerceptKind` kernel set (a stable string name per kernel plus a name-resolution constructor, so a world's DATA can declare which affordance percepts it runs), isolated to `crates/sim/src/affordance_percept.rs` and byte-neutral; (2) FLAG the deeper data-expressible-derivation form (a new kernel as data) as a sibling follow-on. The kernels already read physics, never a kind id, so they are alien-clean on the input side.

## What was framed, and how

Per the owner's standing requirement, the framing was tested blind before any code: the section-11 input-bias smoke test (fail-closed, strongest model at maximum reasoning) on the panel's own construction, then the section-10 blind framing panel (six isolated panelists across three agent types and three models), each seeing only the guiding principles, the neutral mechanism facts, and the raw statement, with no author or owner conclusion and no hint of a suspected flaw.

The smoke test first BLOCKED the construction: it caught that the neutral facts hid the downstream coupling (the percept scalars feed an evolved controller and a discovery loop that propose and score actions, so the percept set partly bounds the emergent affordance space) and that the question set never named that failure mode. The construction was corrected (the downstream fact folded into the neutral packet, a fifth question added naming the downstream-bounding seam, the leading analogies stripped) and the smoke test then returned CLEAR. Only then did the panel run.

The raw statement put to the panel: the kernel set stays a fixed closed in-code set (the mechanism), a world's DATA selects and orders which kernels its beings perceive by naming each with a stable string id, and (the load-bearing claim) which affordance percepts a world's beings sense is therefore world-declared DATA, in the same class as which material axes or sensory channels a world declares, not an outcome that must emerge.

## The panel result: unanimous, and not where the author looked

All six panelists returned `significant-flaw-fixable`, converging independently on the same seam. The string-id selection path (the literal assignment) is clean and Principle-11-compliant, the membership-as-data pattern the function-law catalogue already uses. The defect is the load-bearing equivalence: "which affordance percepts a world runs is world-declared data, in the same class as which material axes a world declares" overclaims, and the two existing kernels prove it. The seams live in the CONTENTS of the kernel set and in the reference, not in the string id.

Three findings, each verified against source before it is trusted (Prime Directive 1):

**1. The pierce kernel is a composite that authors a Terran purpose (Principle 8, admit-the-alien).** `AffordancePerceptKind::Sharpness` runs the compose `Pierce` capability over a held tool's contact-area geometry and material hardness (`affordance_percept.rs:150-173`). Verified: it is a multi-axis composite. It hands the being "this is a good stabber" as a pre-computed primitive percept rather than letting piercing-salience EMERGE from primitives (contact area, hardness, applied stress) composed by the ideation loop. A single-axis magnitude reading is a clean floor primitive; a purpose-laden composite pre-bundled as a percept imposes an affordance from outside the simulation and keys on a Terran tool-user interaction.

**2. The fracture reference is perceiver-independent (admit-the-alien).** `AffordancePerceptKind::FracturePotential` normalizes `mat.fracture_strength` against a `reference_stress`. Verified: the reference is one value on the embodiment (`emb.affordance_refs`, `runner.rs:4963`), shared by every being, and its reserved basis keys it on "the contact stress an ORDINARY being delivers" (`calibration/reserved.toml:2320`), not the perceiving being's own delivered stress. So a mite and a giant sense identical breakability. Correction to the panel, which read the reference as "baked into code": the VALUE is reserved-with-basis in the manifest (`status = "reserved"`, fail-loud), so it is compliant on the value-authoring line. The surviving seam is that it is per-world and not keyed on the perceiver's own data, so an alien body cannot differ as data even within the existing kernel.

**3. The closed kernel TYPE-set caps the emergent affordance space, and "out of scope" prose is not the project's flagging discipline (Principle 8, value-authoring line).** The kernel set is the translation layer between floor physics and the whole downstream discovery space: the controller and the ideation loop can only ever propose actions built from the percepts the set contains (`discovery.rs:74-84`, verified). Two kernels today cap the reachable technique tree by an in-code enumeration rather than by the floor's richness or by selection. A photosynthetic, mana, or redox being cannot express its native affordances as data; it needs a new kernel, which is a code change. Flagging this in one clause of prose is honest about the limit but is not the project's actual tracked-limitation discipline (an R-XXX blockquote, a backlog entry, a Part 62 slot).

**4. A missing invariant: the string id must stay a resolution-only key (the template case).** All six panelists raised it. Verified the downstream is already scalar-by-slot: `discovery.rs:82` keys candidates on the channel INDEX, value-blind, and `discovery.rs:61` states there is no coded primitive-to-affordance pairing (Principle 8). So nothing branches on a name today. But the new naming path introduces a string that must be guaranteed to resolve once to an opaque slot and never be read downstream as a semantic label, or the name becomes a high-level fact driving behaviour (the template-case back door). The guard is cheap and should be stated and tested, not left implicit.

## The corrected framing (survives my own check against the principles)

Split the catalogue by ALTITUDE, not by "selection now versus deeper-derivation later":

- **Tier A (sound, no change):** the kernel derivation (a physical quantity to a [0,1] scalar) stays fixed Rust, the function-law-catalogue precedent.
- **Tier B (the real Principle-11 build):** a parametrized single-axis reduction, "sense a data-named floor axis against a reference that reads from the PERCEIVER's own data or derives from the floor, normalized to [0,1]," whose membership (which axis, which reference source, slot order) is DATA. Fracture-potential becomes a data instance of this form; a broad class of alien senses over floor axes that already exist becomes a data row rather than a rewrite. The reference keys on the being's own data, closing the perceiver-independence seam, and is reserved-with-basis where the floor cannot yet supply it.
- **Tier C (bar or track):** composite affordances (pierce) are barred from the authored floor set and must emerge from the ideation loop composing primitives, OR, if kept pending the composition substrate, are tracked as a first-class Principle-8 research debt, not a footnote.
- **The invariant:** the string id resolves once to an opaque slot; no downstream consumer branches on the id or the name; stated and tested.
- **The deeper substrate:** the fully data-expressible derivation form (a composable algebra over floor axes so a new affordance formula is data) is the honest open research item, tracked in the real discipline.

## The decision this forces (surfaced, not assumed)

The panel's reframing revises the gate's confirmed scope, so per Prime Directive 5 I do not proceed on my original framing. I surface the fork for the gate to rule, mirroring how the slice-1 reach-wire framing was surfaced:

- **Fork (a): adopt the corrected framing now.** Build Tier B (the parametrized single-axis reduction registry with a perceiver-keyed reference), relocate or bar the pierce composite (Tier C), add the opaque-id invariant, and open the R-XXX for the composition substrate. Cleaner and alien-clean and Principle-8-clean at the percept layer, but larger, and barring or relocating pierce and moving to a per-being reference changes behaviour where those kernels are wired, so byte-neutrality holds only for the untouched reduction path, not for the pierce relocation.
- **Fork (b): keep the narrow assignment now, track the seams as first-class items.** Build only the string-name selection path (byte-neutral, no live caller), and do NOT describe it as making the affordance space world-declared or alien-clean. Immediately open the three seams (the pierce composite P8 debt, the perceiver-independent reference, the type-set ceiling) plus the composition substrate as tracked R-XXX items in the real discipline, each to be built as its own gated piece.

My recommendation is a hybrid weighted toward (b): build the clean, byte-neutral string-name selection path as the isolated piece it was scoped to be, and open the three seams and the composition substrate as first-class R-XXX items now rather than as prose, because the panel is right that an "out of scope" aside is how a real Principle-8 collision goes invisible. Tier B (the parametrized reduction with a perceiver-keyed reference) and Tier C (barring the composite) are the deeper builds; the gate should scope them as their own gated pieces rather than fold them into this one. The gate rules.
