# Units / R-UNITS-PIN: kickoff (the fixed-point composite compute, retiring the sigma sentinel and the dev-fixtures interim together)

This is a DOC-ONLY kickoff, the bridge PR for the units follow-on the gate directed after signing off the fundamentals-home (PR #125). It scopes the seam and states the discipline; it authors no mechanism and moves no value. The predecessor #125 merges once this bridge is open (the bridge rule). Opened off current `main`.

## The seam

The fundamentals-home (#125) landed the closed CODATA table (`crates/units/src/fundamentals.rs`) and marked the Stefan-Boltzmann sigma as a derive-target: `metabolism.stefan_boltzmann` was retired from `calibration/reserved.toml` (status reserved, value emptied, basis pointing at `civsim_units::fundamentals::STEFAN_BOLTZMANN` with the formula `2 * pi^5 * k_B^4 / (15 * h^3 * c^2)`). But the derivation is not yet COMPUTED: the table carries the composite's value as a `&'static str` decimal alongside its formula and its fundamentals list, and the drift-check that guards it is an f64 test-only comparison, not the canonical-path compute. Two interim props hold the boot up in the meantime: the sigma sentinel's retirement is inert (the value string is empty, the basis is a pointer), and `calibration/profiles/dev-fixtures.toml` KEEPS `metabolism.stefan_boltzmann = 0.0000000567` as a labeled interim so `run_world` does not panic. Both must come down together, and neither can until sigma is computed from the fundamentals on the canonical fixed-point path.

The reason it is not already computed is a substrate seam, not an oversight. The composite compute reads the fundamentals, and the fundamentals underflow Q32.32. The canonical `Fixed` is Q32.32 with a smallest positive magnitude near 2.3e-10; sigma itself (5.670e-8) sits above that floor, but the fundamentals it derives FROM do not: k_B is 1.38e-23, h is 6.626e-34, the elementary charge is 1.6e-19. None of the three is representable in raw Q32.32, so `k_B^4 / (h^3 * c^2)` cannot be evaluated term by term in the canonical scale without collapsing to zero. Computing the composite deterministically in fixed-point requires a representation for magnitudes that underflow the canonical epsilon. That representation is a substrate-design choice with real seams (per-quantity scale versus a scaled-exponent pair, where the rounding happens, how the intermediate products stay in range, how the result rescales back to sigma's canonical quantity scale), which is why this arc is frame-blinded before a line of it is written.

## What the arc opens (scope, not mechanism)

The units crate already carries the mechanism this leans on: a per-quantity fixed-point scale (`QuantityDef::scale_bits`), a scale-derivation rule from a declared envelope (`derive_scale_bits`), and a deterministic rescale between scales (`rescale_bits`). The arc's job is to use that substrate (or extend it where it cannot reach) to compute the composites, not to invent a parallel one. Scope, each piece to be built only after the frame-blind clears and the gate rules:

1. The FIXED-POINT COMPOSITE COMPUTE. A deterministic evaluation of a composite's formula from its fundamentals, in the units crate's per-quantity scaled representation rather than raw Q32.32, so k_B, h, e, and the rest are each held at a scale where they are representable and the product `k_B^4 / (h^3 * c^2)` stays in range through its intermediates and rescales to sigma's canonical quantity scale. This is the scaled-exponent (or equivalent per-quantity-scale) representation the R-UNITS-PIN flag anticipates. The representation is the design choice under audit; the compute is its first consumer.

2. RETIRE THE SIGMA SENTINEL AND THE DEV-FIXTURES INTERIM TOGETHER. Once sigma is computed from the fundamentals on the canonical path, the retired-and-empty `metabolism.stefan_boltzmann` in `reserved.toml` and the interim `metabolism.stefan_boltzmann = 0.0000000567` in `dev-fixtures.toml` both come down in one move, with the metabolism kernel reading the computed composite. Neither comes down before the compute exists (removing the dev-fixtures value today panics `run_world` at the metabolism boot, the empirically-verified input-audit catch that kept it as a labeled interim in #125).

3. RE-ARM THE MIRROR COVERAGE (the gate's explicit condition). The `mirror_calibrated_boot` integration test's calibrated-profile determinism and worker-invariance checks were inerted in #122's CI fix: the Err branch treats a `category = derivable` derive-target block as expected (prints a note and passes) rather than panicking, because the calibrated boot reads sigma via `from_manifest` and sigma is now a derive-target. Once sigma DERIVES on the canonical path, that inerting is lifted: the calibrated boot boots on the computed sigma, and the determinism and worker-invariance assertions are re-armed so the inerted checks are not forgotten. This is a named scope item, not an afterthought.

## The first act (the gate's forward-note)

The arc's FIRST act, before the compute is built, is a cross-check of the PARSED formula against the drift-check reference. Today the drift-check validates the hardcoded decimal copy of sigma against an f64 recomputation; it does not validate that the `formula` STRING (`2 * pi^5 * k_B^4 / (15 * h^3 * c^2)`) is what the compute evaluates. When the compute begins to read and evaluate that string, the string it computes from must be cross-checked against the drift-check reference at the outset, so the parsed formula cannot silently diverge from the value the table ships and the test guards. A formula the compute reads that disagrees with the drift-check reference is a defect caught on entry, not after the compute is wired.

## The discipline (why this is a substrate-design arc, frame-blind first)

No mechanism is authored in this PR. The fixed-point representation of magnitudes that underflow Q32.32 is a substrate-design choice with real seams, so it is FRAME-BLINDED before code: the section-11 input-bias smoke on the framing construction first (fail-closed, Opus max), then the section-10 blind panel against the principles, then the gate rules on the representation before a line is written. The build that follows runs the mandatory section-9 five-lens audit and holds byte-neutrality against the four canonical `run_world` pins throughout: the compute changes what sigma's PROVENANCE is (a derivation instead of an authored decimal), and the intent is that it does NOT change sigma's VALUE at the canonical scale within fixed-point tolerance, so the pins stay identical. Where the computed sigma cannot reproduce the retired decimal to within fixed-point tolerance, that gap is surfaced to the gate as a reserved tolerance with its basis (the fixed-point epsilon at sigma's quantity scale), never papered over.
