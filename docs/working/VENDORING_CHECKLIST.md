# The vendoring checklist (the proof-of-fetch protocol for a cited [M] data artifact)

> **THE STANDING RULE, READ FIRST (a session-start requirement for every fetch sub-agent, AGENTIC_ADDENDUM.md section 12).** If you produce a load-bearing number, VENDOR ITS SOURCE at fetch time: download the actual document or data file, SHA256 it, and hold it here behind the manifest. A URL citation is NOT provenance, because a link rots and the number is then a claim with no held witness. Read the primary's figures and tables rather than the abstract, and carry each value's anchor (page, figure, table), a dual-channel agreement where the standard needs it, and its SCOPE (the regime it applies to). A number whose provenance is a hyperlink has not been fetched.

Every measured or externally-sourced data column ([M], and the compute-once cited tier) enters the repo as a vendored artifact behind a manifest. The manifest is the artifact's contract; a CI gate is the classifier of last resort, and the point of this checklist is that classification happens at FETCH TIME, by the agent who knows what the artifact is, not later by a gate that only knows what it is not. The κ_R arc's covalent-radius column shipped for two commits with the floor-provenance gate red because its block kind was never declared at fetch time; that failure class closes here.

For each vendored artifact, the manifest (or the data file's header) declares:

- **name / id**: the artifact's key.
- **citation**: the primary source, verified at the primary before the value is entered (never a secondary paraphrase for a load-bearing constant).
- **sha256**: the checksum of the fetched bytes (the receipt; immutable forever, per the recipe/receipt rule). SHA256 is primary from 2026-07-17, because md5 collisions are practically constructible and a collision-forgeable receipt is not tamper-evident, which is the one property a receipt exists to have. An md5 may ride as a legacy secondary (the Byerlee 1978 import carries one).
- **recipe**: the query provenance (the exact POST body / DOI / fetch parameters), so the fetch is reproducible and the dry-run battery can reconstruct it byte-for-byte.
- **block kind**: the TOML block name the data uses, and it MUST NOT be a reserved floor kind (`[[element]]`, `[[axis]]`, `[[substance]]`) unless the artifact truly IS a floor axis carrying the real/fantasy authorship split. A cited [M] import never participates in that axis: it is an immutable transcription of its source, and procgen values for an authored entity come from the estimator tier at runtime, never written into the citation file (co-location laundering). Use the cited-data-column idiom instead (the Shannon ionic radii and the Pyykkö covalent radii both use `[[radius]]`; Badger uses `[[pair]]`; the optical constants `[[species]]`; the ionization ladder `[[ladder]]`; the TM-oxide energies `[[oxide]]`).
- **provenance idiom**: how the column carries provenance. A floor manifest declares per-entry `real = "..."` / `fantasy = "..."`. A cited data column carries a header citation and is out of floor-Element policing. State which, so the gate and the reader agree.
- **grade uniformity**: whether the source is single-grade or MIXED-grade. If a source mixes grades (e.g. the Pyykkö column: light rows are fits to measured bond distances, heavy actinide-and-beyond rows are relativistic computed estimates), declare the measured-versus-computed boundary from the source's own data-source notes, and add a loader guard that flags reads above the boundary. Per-entry grade annotation is required only where a source mixes grades AND a consumer reads the mixed region; otherwise the header boundary plus the loader guard suffice. (Flagged-for-later: Draine's "astronomical silicate" is a constructed object rather than a lab measurement, so if the mixed-grade vocabulary ever wants a second customer, that optical file is it.)

The `*_provenance_test.py` battery then checks each artifact offline, with no network. **What each test actually proves varies, and this document used to overstate it uniformly.** Audited 2026-07-19 against all eight:

| Test | What it PROVES | What it does NOT prove |
| --- | --- | --- |
| `janaf` | Artifact integrity: 34 held tables hash against their receipts. Plus transcription of the Lodders fingerprints. | That the condensation solver agrees with Lodders. That belongs in a separate system test with Lodders held out. |
| `bhac15` | That the committed wall grid was extracted from the held BHAC15 bytes. | That BHAC15 stellar evolution is correct. |
| `aesopus` | POST-recipe reconstruction and manifest consistency. | Physical validation. The gas `.dat` files are not hashed against a receipt here. |
| `condensation_mu` | Transcription of the AGSS09 and Robie-Hemingway rows; that the ice sample points were generated from the committed coefficients. | Agreement with Murphy-Koop. Altering the coefficients and regenerating the points passes: it is a generated-column consistency check, not a validation. |
| `convection_scaling` | Analytic verification of `27 pi^4 / 4`, `pi / sqrt(2)` and `2^(-4/3)`, which is real. Plus transcription of the fingerprint dictionary. | Empirical Nusselt-Rayleigh accuracy. |
| `gruneisen` | Custody of three citation-plus-witness sources; transcription of seven rows. | Independent agreement. Its `1.803` Slater expectation is BACK-SOLVED from the same `K' = 3.94` it checks, and its applicability sentinel reads the `slater_gamma_applicable` flag on the very rows under test. |
| `rayleigh_eigenvalue` | Closed-form analytic verification of the free-free case. | The non-analytic eigenvalues, which are transcribed rather than recomputed. |
| `disk_arc_literature` | Receipt completeness and schema shape across 31 sources. | That any extract or grade is scientifically correct. |

**The honest summary: these are CUSTODY, TRANSCRIPTION and ANALYTIC-VERIFICATION checks.** Custody proves the bytes we hold are the bytes we fetched. Transcription proves the number in our column matches the number in the held source. Analytic verification proves a closed form is computed correctly. All three are worth having and all three catch real drift. None of them establishes that the source is RIGHT, and a transcription check that reads its expectation from the file under test does not even establish independence.

Three are worth rebuilding as genuine validation, and are recorded rather than quietly left: the ice correlation against IAPWS R14-08(2011) over 130 to 220 K; the Slater applicability classification against held-out thermodynamic gamma measurements instead of the row's own flag; and the non-analytic Rayleigh eigenvalues against an independent neutral-stability solver.

## Vendored artifacts

A DOCUMENT witness (a paper held so an equation or a table can be re-read) declares the same fields as a data column, with two of them answered by their absence: it carries no TOML block kind and no per-entry provenance idiom, because it is not a data column. It is held SLIMMED to the pages that were read, so custody is a witness rather than a whole PDF nobody opens again, and the entry records what was kept and what was dropped so the slim is never mistaken for the whole document.

### `mixing_henke2016`: Henke, Gail and Trieloff 2016, the numerical benchmark that settles the multiphase mixing rule

- **name / id**: `mixing_henke2016`. Held at `~/.claude/vendored-sources/rock_conductivity_mixing/`.
- **citation**: Henke, S., Gail, H.-P. and Trieloff, M., 2016, "Thermal evolution and sintering of chondritic planetesimals. III. Modelling the heat conductivity of porous chondrite material", Astronomy and Astrophysics 589, A41, DOI 10.1051/0004-6361/201527687. Journal reference verified at Crossref (volume 589, page A41, May 2016) rather than assumed from the preprint. Load-bearing content: Section 3.3 with equations (13) through (17), Table 3 (the six-component mineral compositions with their volume fractions and conductivities), Table 4 (the binary nickel-iron plus forsterite series), and Table 6 (the numerical-versus-Bruggeman comparison at each porosity).
- **status**: RETRIEVED, arXiv author preprint `arXiv:1602.00292v1`, NOT the publisher's version of record. The A&A open-access copy was attempted at `https://www.aanda.org/articles/aa/pdf/2016/05/aa27687-15.pdf` and at the DOI, and both returned HTTP 500 from the publisher's own server on 18 July 2026. The entry declares the preprint status rather than claiming a version it does not hold; a value read here is a preprint reading until the typeset version is retrieved.
- **sha256 (fetch receipt, the full 19-page preprint as fetched)**: `f62bf64fbcd97dc0111c250c214343dbd77af3a2052f584dc69365e4a8870935`, 1,128,030 bytes, 19 pages.
- **sha256 (the held slim)**: `f9bcb3be8f7101955f1bcb61a206607ed29a45e6a1338e837f46fb9bb27d292e`, 119,223 bytes, 6 pages, file `henke_gail_trieloff_2016_AA589_A41_SLIM_p1_p5_p7-9_p19.pdf`.
- **recipe**: `curl -L "https://arxiv.org/pdf/1602.00292"` (HTTP 200, `application/pdf`), then keep PDF pages 1, 5, 7, 8, 9 and 19 (zero-based indices 0, 4, 6, 7, 8, 18) with `pypdf.PdfWriter`. Both hashes are reproducible from that pair of steps.
- **archived_url**: `https://web.archive.org/web/20260719002840/https://arxiv.org/pdf/1602.00292`, captured fresh on 19 July 2026. A link is not provenance, so the archive stands behind the receipt rather than in place of it.
- **kept in the slim**: PDF page 1, the identification: title, the three authors, the Heidelberg affiliations, and the arXiv stamp. PDF page 5, Table 3, the typical mineral composition of H, L, LL and EH chondrites, giving each component's volume fraction `f_min` and its room-temperature conductivity `K`, which is the input census every rule in this arc was scored against. PDF page 7, Table 4, the binary nickel-iron and forsterite series at six volume fractions with the numerically solved conductivity at three grid resolutions beside the effective-medium column, and Table 5, the ten-seed scatter that fixes the numerical noise floor. PDF page 8, THE load-bearing page for the rule itself: equation (13) the geometric mean cited to Clauser and Huenges 1995, the authors' verdict on it, equation (14) Berryman's function and equations (15) and (16) the resulting upper and lower bounds, and Figure 3's caption. PDF page 9, equation (17), the Bruggeman self-consistent rule, and Table 6, the numerical-versus-Bruggeman comparison for all four chondrite classes across porosity 0.00 to 0.30. PDF page 19, the reference list, so the citations to Bruggeman (1935), Berryman (1995), Clauser and Huenges (1995), Horai and Baldridge (1972) and Yomogida and Matsui (1983) can be cashed inside the held witness.
- **dropped from the slim**: PDF pages 2 to 4 (the introduction and the pure-component conductivity tables 1 and 2), page 6 (the homogenization argument and the numerical method), pages 10 to 18 (the porous and sintered granular models, the ball-packing appendices, the temperature dependence, and the thermal-evolution application). None carries a mixing rule or the census that scores one. The full document is recoverable from the archived URL and checkable against the fetch receipt above.
- **block kind**: none. This is a document witness, not a data column, so it declares no TOML block and never touches floor-Element policing.
- **provenance idiom**: the held slim plus the two hashes plus the archived URL. The rule read from it is stated at its implementation site in `crates/materials/src/conductivity.rs` with its equation number.
- **grade uniformity**: MIXED in a way that matters, and the boundary is the reason this artifact was fetched. Table 3's per-component conductivities are MEASURED values compiled from Horai and Baldridge (1972) and Yomogida and Matsui (1983); Table 4's and Table 6's "numerical" columns are COMPUTED, a finite-difference solution of the heat conduction equation on a random voxel mixture, with a stated seed-to-seed scatter; the Bruggeman column is an ANALYTIC closure evaluated on the same census. The rule was selected by scoring the analytic closures against the COMPUTED column, so what this artifact licenses is a claim about agreement with a direct numerical solution, never a claim of agreement with a laboratory measurement of a rock.
- **read channels**: text extraction (`pdftotext -layout`) plus an INDEPENDENT RECOMPUTATION: equation (17) was re-derived symbolically to confirm it is the standard symmetric Bruggeman form `sum_i f_i (K_i - K) / (K_i + 2K) = 0`, and the rule was then re-solved by bisection on the Table 3 censuses and reproduced the paper's own Bruggeman column to better than 0.02 W/(m*K) in all four chondrite classes. That recomputation is the dual channel for this artifact, because the load-bearing content is an implicit equation whose transcription cannot be checked by re-reading it.
