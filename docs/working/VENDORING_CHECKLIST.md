# The vendoring checklist (the proof-of-fetch protocol for a cited [M] data artifact)

> **THE STANDING RULE, READ FIRST (a session-start requirement for every fetch sub-agent, AGENTIC_ADDENDUM.md section 12).** If you produce a load-bearing number, VENDOR ITS SOURCE at fetch time: download the actual document or data file, SHA256 it, and hold it here behind the manifest. A URL citation is NOT provenance, because a link rots and the number is then a claim with no held witness. Read the primary's figures and tables rather than the abstract, and carry each value's anchor (page, figure, table), a dual-channel agreement where the standard needs it, and its SCOPE (the regime it applies to). A number whose provenance is a hyperlink has not been fetched.

Every measured or externally-sourced data column ([M], and the compute-once cited tier) enters the repo as a vendored artifact behind a manifest. The manifest is the artifact's contract; a CI gate is the classifier of last resort, and the point of this checklist is that classification happens at FETCH TIME, by the agent who knows what the artifact is, not later by a gate that only knows what it is not. The κ_R arc's covalent-radius column shipped for two commits with the floor-provenance gate red because its block kind was never declared at fetch time; that failure class closes here.

For each vendored artifact, the manifest (or the data file's header) declares:

- **name / id**: the artifact's key.
- **citation**: the primary source, verified at the primary before the value is entered (never a secondary paraphrase for a load-bearing constant).
- **sha256**: the checksum of the fetched bytes (the receipt; immutable forever, per the recipe/receipt rule). SHA256 is primary from 2026-07-17, because md5 collisions are practically constructible and a collision-forgeable receipt is not tamper-evident, which is the one property a receipt exists to have. An md5 may ride as a legacy secondary (the Byerlee 1978 import carries one).
- **recipe**: the query provenance (the exact POST body / DOI / fetch parameters), so the fetch is reproducible and the dry-run battery can reconstruct it byte-for-byte.
- **block kind**: the TOML block name the data uses, and it MUST NOT be a reserved floor kind (`[[element]]`, `[[axis]]`, `[[substance]]`) unless the artifact truly IS a floor axis carrying the real/fantasy authorship split. A cited [M] import never participates in that axis: it is an immutable transcription of its source, and procgen values for an authored entity come from the estimator tier at runtime, never written into the citation file (co-location laundering). Use the cited-data-column idiom instead (the Shannon ionic radii and the Pyykkö covalent radii both use `[[radius]]`; Badger uses `[[pair]]`; the optical constants `[[species]]`; the ionization ladder `[[ladder]]`; the TM-oxide energies `[[oxide]]`).
- **provenance idiom**: how the column carries provenance. A floor manifest declares per-entry `real = "..."` / `fantasy = "..."`. A cited data column carries a header citation and is out of floor-Element policing. State which, so the gate and the reader agree.
- **grade uniformity**: whether the source is single-grade or MIXED-grade. If a source mixes grades (e.g. the Pyykkö column: light rows are fits to measured bond distances, heavy actinide-and-beyond rows are relativistic computed estimates), declare the measured-versus-computed boundary from the source's own data-source notes, and add a loader guard that flags reads above the boundary. Per-entry grade annotation is required only where a source mixes grades AND a consumer reads the mixed region; otherwise the header boundary plus the loader guard suffice. (Flagged-for-later: Draine's "astronomical silicate" is a constructed object rather than a lab measurement, so if the mixed-grade vocabulary ever wants a second customer, that optical file is it.)

The dry-run byte-equality battery (the `*_provenance_test.py` pattern) then reconstructs each fetch from its recipe and asserts byte-equality against the receipt, with no network, so a drifted transcription fails the build.

## Vendored artifacts

A DOCUMENT witness (a paper held so an equation or a table can be re-read) declares the same fields as a data column, with two of them answered by their absence: it carries no TOML block kind and no per-entry provenance idiom, because it is not a data column. It is held SLIMMED to the pages that were read, so custody is a witness rather than a whole PDF nobody opens again, and the entry records what was kept and what was dropped so the slim is never mistaken for the whole document.

### `flexure_tafi`: Jha, Harry and Schutt 2017 (TAFI), the flexure Green's functions and Table 1

- **name / id**: `flexure_tafi`. Held at `~/.claude/vendored-sources/flexure_tafi/`.
- **citation**: Jha, S., Harry, D. L. and Schutt, D. L., 2017, "Toolbox for Analysis of Flexural Isostasy (TAFI): A MATLAB toolbox for modeling flexural deformation of the lithosphere", Geosphere 13(5), 1555-1565, DOI 10.1130/GES01421.1. Gold open access. Load-bearing content: equations (1) through (7) and Table 1, printed pages 1556-1557.
- **status**: RETRIEVED, publisher's version of record. The PDF is the GSA typeset final (Adobe InDesign CC 2017, created 11 September 2017, matching the paper's own printed "Final version posted 11 September 2017"), so it is the version of record rather than an accepted manuscript. The DOI itself resolves to a Cloudflare 403 at `pubs.geoscienceworld.org`, so the bytes came from the author-hosted copy on the third author's site, which is the same typeset document.
- **sha256 (fetch receipt, the full 11-page document as fetched)**: `4abc31ee08633de676313a600f4d4bf1a34b70ce131dd20dee2d1a078bc8f009`, 2,210,652 bytes, 11 pages. This is the receipt for what was retrieved, and it matches the hash already on record in `docs/working/POINT_LOAD_LENGTH_SCALE_FETCH.md` byte for byte, so two independent fetches on two dates read identical bytes.
- **sha256 (the held slim)**: `b0733092524e96ce209b564725855deda9493eba31035b2ad8e710e0371a9240`, 942,728 bytes, 4 pages, file `jha_harry_schutt_2017_tafi_geosphere_SLIM_p1555-1557_p1564.pdf`.
- **recipe**: `curl -L "https://derekschutt.wordpress.com/wp-content/uploads/2018/01/flexural_modeling_geosphere_2017.pdf"` (HTTP 200, `application/pdf`), then keep PDF pages 1, 2, 3 and 10 (zero-based indices 0, 1, 2, 9) with `pypdf.PdfWriter`. Both hashes above are reproducible from that pair of steps.
- **archived_url**: `https://web.archive.org/web/20260718230721/https://derekschutt.wordpress.com/wp-content/uploads/2018/01/flexural_modeling_geosphere_2017.pdf`, captured 18 July 2026 and verified to return HTTP 200 with `application/pdf`. A link is not provenance, so the archive stands behind the receipt rather than in place of it.
- **kept in the slim**: printed page 1555 (PDF 1), the identification: title, the three authors, the Colorado State University affiliation, journal, volume, issue, page range, DOI, the citation block, the open-access statement, and the submission and posting dates. Printed page 1556 (PDF 2), the governing equations (1) and (2) in Cartesian and radial form and the flexural rigidity (3). Printed page 1557 (PDF 3), THE load-bearing page: equations (4), (5), (6) and (7), and Table 1 with its four load-model rows, its column headings, its variable note, and its two source footnotes. Printed page 1564 (PDF 10), the reference list, so Table 1's footnote citations to Brotchie and Silvester (1969) and Turcotte and Schubert (2002) can be cashed inside the held witness.
- **dropped from the slim**: printed pages 1558-1563 (PDF 4-9), the program design, the GUI description, the function tables, the worked Aleutian trench example, and the Parker gravity algorithm; and printed page 1565 (PDF 11), the closing matter. None carries a flexure equation or a flexural-parameter definition. The full document is recoverable from the archived URL and checkable against the fetch receipt above.
- **block kind**: none. This is a document witness, not a data column, so it declares no TOML block and never touches floor-Element policing.
- **provenance idiom**: the held slim plus the two hashes plus the archived URL. Every value read from it is quoted in `docs/working/PIPELINE_FETCHES.md` section 1 with its equation or table number.
- **grade uniformity**: single-grade. The artifact is a published paper's analytic equations, transcribed rather than measured, so the measured-versus-computed boundary does not arise.
- **read channels**: text extraction (`pdftotext -layout`) and a visual render (`pdftoppm`) of printed page 1557 at 450 dpi, with the Table 1 point-load flexural-parameter cell re-rendered at 900 dpi. The dual channel is the standard for this artifact because the load-bearing content is a table cell whose exponent and denominator decide a length scale.
- **corroborating artifact, not vendored here**: the toolbox source itself, `github.com/sumantjha/TAFI_v1.0`, whose `TAFI_v1.0/Geodynamic_functions/Geodynamic_Utilities/flexparam.m` is the authors' own machine-readable statement of the same quantity. It is a separate artifact under separate terms and is cited rather than held.
