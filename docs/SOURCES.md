# Sources: the centralized bibliography

GENERATED by `scripts/gen_sources.py`. Do not edit by hand.

Every source the project holds or witnesses, with its citation, its checksum, where the bytes are,
and whether an archive snapshot stands behind the link. This is the single bibliography for the
vendored population: `sources/registry.toml` (hand-maintained, where a new fetch lands) and
`sources/mirrored.toml` (generated from the per-directory manifests) read as one namespace.

Custody is one of three. `in_repo`: bytes held in-tree, checksum verifiable with no network.
`witness`: bytes not held, the checksum is a re-fetch receipt and the archive URL is the public
witness. `external`: bytes in local custody outside the repo, checksum plus archive snapshot.
The licence decides which is permitted; see `docs/working/FETCH_PIPELINE_PLAN.md` section 3.
